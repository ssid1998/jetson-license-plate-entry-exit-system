\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md147}{}\doxysection{\texorpdfstring{Building the Project from Source}{Building the Project from Source}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md147}
Provided with the repo is a library of Tensor\+RT-\/accelerated deep learning networks for image recognition, object detection with localization (i.\+e. bounding boxes), and semantic segmentation. This inferencing library ({\ttfamily libjetson-\/inference}) is intended to be built \& run on the Jetson, and includes support for both C++ and Python. ~\newline


Various pre-\/trained DNN models are automatically downloaded to get you up and running quickly. It\textquotesingle{}s also setup to accept customized models that you may have trained yourself, including support for Caffe, Tensor\+Flow UFF, and ONNX.

\doxylink{requirements_The}{The} latest source can be obtained from \href{http://github.com/dusty-nv/jetson-inference}{\texttt{Git\+Hub}} and compiled onboard Jetson Nano, Jetson TX1/\+\+TX2, and Jetson AGX Xavier once they have been \doxysectlink{md_jetson-inference_2docs_2jetpack-setup}{flashed with Jet\+Pack}{0} or setup with the pre-\/populated \href{https://developer.nvidia.com/embedded/learn/get-started-jetson-nano-devkit\#write}{\texttt{SD card image}} for Jetson Nano.\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md148}{}\doxysubsubsection{\texorpdfstring{Quick Reference}{Quick Reference}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md148}
Here\textquotesingle{}s a condensed form of the commands to download, build, and install the project:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}

\DoxyCodeLine{\$\ sudo\ apt-\/get\ install\ git\ cmake\ libpython3-\/dev\ python3-\/numpy}

\DoxyCodeLine{\$\ cd\ jetson-\/inference}
\DoxyCodeLine{\$\ mkdir\ build}
\DoxyCodeLine{\$\ cd\ build}
\DoxyCodeLine{\$\ cmake\ ../}
\DoxyCodeLine{\$\ make\ -\/j\$(nproc)}
\DoxyCodeLine{\$\ sudo\ make\ install}
\DoxyCodeLine{\$\ sudo\ ldconfig}

\end{DoxyCode}
 Below we will go through each step and discuss various build options along the way.\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md149}{}\doxysubsubsection{\texorpdfstring{Cloning the Repo}{Cloning the Repo}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md149}
To download the code, navigate to a folder of your choosing on the Jetson. First, make sure git and cmake are installed:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ sudo\ apt-\/get\ update}
\DoxyCodeLine{\$\ sudo\ apt-\/get\ install\ git\ cmake}

\end{DoxyCode}


Then clone the {\ttfamily jetson-\/inference} project:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ git\ clone\ https://github.com/dusty-\/nv/jetson-\/inference}
\DoxyCodeLine{\$\ cd\ jetson-\/inference}
\DoxyCodeLine{\$\ git\ submodule\ update\ -\/-\/init}

\end{DoxyCode}


Remember to run the {\ttfamily git submodule update -\/-\/init} step (or clone with the {\ttfamily -\/-\/recursive} flag).\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md150}{}\doxysubsubsection{\texorpdfstring{Python Development Packages}{Python Development Packages}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md150}
\doxylink{requirements_The}{The} Python functionality of this project is implemented through Python extension modules that provide bindings to the native C++ code using the Python C API. While configuring the project, the repo searches for versions of Python that have development packages installed on the system, and will then build the bindings for each version of Python that\textquotesingle{}s present (e.\+g. Python 2.\+7, 3.\+6, and 3.\+7). It will also build numpy bindings for versions of numpy that are installed.

By default, Ubuntu comes with the {\ttfamily libpython-\/dev} and {\ttfamily python-\/numpy} packages pre-\/installed (which are for Python 2.\+7). Although the Python 3.\+6 interpreter is pre-\/installed by Ubuntu, the Python 3.\+6 development packages ({\ttfamily libpython3-\/dev}) and {\ttfamily python3-\/numpy} are not. These development packages are required for the bindings to build using the Python C API. ~\newline


So if you want the project to create bindings for Python 3.\+6, install these packages before proceeding:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ sudo\ apt-\/get\ install\ libpython3-\/dev\ python3-\/numpy}

\end{DoxyCode}


Installing these additional packages will enable the repo to build the extension bindings for Python 3.\+6, in addition to Python 2.\+7 (which is already pre-\/installed). Then after the build process, the \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.inference.html}{\texttt{{\ttfamily jetson.\+inference}}} and \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.utils.html}{\texttt{{\ttfamily jetson.\+utils}}} packages will be available to use within your Python environments.\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md151}{}\doxysubsubsection{\texorpdfstring{Configuring with CMake}{Configuring with CMake}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md151}
Next, create a build directory within the project and run {\ttfamily cmake} to configure the build. When {\ttfamily cmake} is run, a script is launched (\href{../CMakePreBuild.sh}{\texttt{{\ttfamily CMake\+Pre\+Build.\+sh}}}) that will install any required dependencies and download DNN models for you.


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference\ \ \ \ \#\ omit\ if\ working\ directory\ is\ already\ jetson-\/inference/\ from\ above}
\DoxyCodeLine{\$\ mkdir\ build}
\DoxyCodeLine{\$\ cd\ build}
\DoxyCodeLine{\$\ cmake\ ../}

\end{DoxyCode}


\begin{quote}
{\bfseries{note}}:\+ this command will launch the \href{../CMakePreBuild.sh}{\texttt{{\ttfamily CMake\+Pre\+Build.\+sh}}} script which asks for sudo privileges while installing some prerequisite packages on the Jetson. \doxylink{requirements_The}{The} script also downloads pre-\/trained networks from web services. \end{quote}
\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md152}{}\doxysubsubsection{\texorpdfstring{Downloading Models}{Downloading Models}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md152}
\doxylink{requirements_The}{The} project comes with many pre-\/trained networks that can you can choose to have downloaded and installed through the {\bfseries{Model Downloader}} tool (\href{../tools/download-models.sh}{\texttt{{\ttfamily download-\/models.\+sh}}}). By default, not all of the models are initially selected for download to save disk space. You can select the models you want, or run the tool again later to download more models another time.

When initially configuring the project, {\ttfamily cmake} will automatically run the downloader tool for you:\+



\begin{quote}
{\bfseries{note}}:\+ for users that are unable to connect to Box.\+com to download the models, a mirror is provided here:\+~\newline
 ~~~~~~~~~~~~~\href{https://github.com/dusty-nv/jetson-inference/releases}{\texttt{{\ttfamily https:\+/\+/\+github.\+com/\+dusty-\/nv/\+jetson-\/inference/\+releases}}} \end{quote}


To run the Model Downloader tool again later, you can use the following commands:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/tools}
\DoxyCodeLine{\$\ ./download-\/models.sh}

\end{DoxyCode}
\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md153}{}\doxysubsubsection{\texorpdfstring{Installing Py\+Torch}{Installing Py\+Torch}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md153}
If you are using Jet\+Pack 4.\+2 or newer, another tool will now run that can optionally install Py\+Torch on your Jetson if you want to re-\/train networks with \doxysectlink{md_jetson-inference_2docs_2pytorch-transfer-learning}{transfer learning}{0} later in the tutorial. This step is optional, and if you don\textquotesingle{}t wish to do the transfer learning steps, you don\textquotesingle{}t need to install Py\+Torch and can skip this step.

If desired, select the Py\+Torch package versions for Python 2.\+7 and/\+or Python 3.\+6 that you want installed and hit {\ttfamily Enter} to continue. Otherwise, leave the options un-\/selected, and it will skip the installation of Py\+Torch.



\begin{quote}
{\bfseries{note}}:\+ the automated Py\+Torch installation tool requires Jet\+Pack 4.\+2 (or newer)~\newline
 ~~~~~~~~~~for other versions, see \href{https://elinux.org/Jetson_Zoo\#PyTorch_.28Caffe2.29}{\texttt{{\ttfamily http:\+/\+/\+e\+Linux.\+org/\+\+Jetson\+\_\+\+Zoo}}} to build from source. \end{quote}


You can also run this tool again later if you decide that you want to install Py\+Torch at another time:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/build}
\DoxyCodeLine{\$\ ./install-\/pytorch.sh}

\end{DoxyCode}


Running these commands will prompt you with the same dialog as seen above.\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md154}{}\doxysubsubsection{\texorpdfstring{Compiling the Project}{Compiling the Project}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md154}

\begin{DoxyCode}{0}

\doxylink{requirements_The}{The} Python bindings for the \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.inference.html}{\texttt{{\ttfamily jetson.\+inference}}} and \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.utils.html}{\texttt{{\ttfamily jetson.\+utils}}} modules also get installed during the {\ttfamily sudo make install} step under {\ttfamily /\+usr/\+lib/\+python\texorpdfstring{$\ast$}{*}/\+dist-\/packages/\+}. If you update the code, remember to run it again.\hypertarget{md_jetson-inference_2docs_2building-repo_autotoc_md155}{}\doxysubsubsection{\texorpdfstring{Digging Into the Code}{Digging Into the Code}}\label{md_jetson-inference_2docs_2building-repo_autotoc_md155}
