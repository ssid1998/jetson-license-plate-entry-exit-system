 \textsuperscript{Web\+App Frameworks}\hypertarget{md_jetson-inference_2docs_2webrtc-server_autotoc_md448}{}\doxysection{\texorpdfstring{Web\+RTC Server}{Web\+RTC Server}}\label{md_jetson-inference_2docs_2webrtc-server_autotoc_md448}
jetson-\/inference includes an integrated Web\+RTC server for streaming low-\/latency live video to/\+from web browsers that can be used for building dynamic web applications and data visualization tools powered by Jetson and edge AI on the backend. Web\+RTC works seamlessly with DNN inferencing pipelines via the \href{aux-streaming.md\#source-code}{\texttt{{\ttfamily video\+Source/\+video\+Output}}} interfaces from jetson-\/utils, which utilizes hardware-\/accelerated video encoding and decoding through GStreamer. It supports sending and receiving multiple streams to/\+from multiple clients simultaneously (without needing to re-\/encode the video for each independent client), and includes a built-\/in webserver frontend for viewing video streams remotely:\+



In this screenshot of full-\/duplex mode, the webcam from a laptop is being streamed to a Jetson over Web\+RTC, where the Jetson decodes it and performs object detection using \doxylink{classdetect_net}{detect\+Net}, before re-\/encoding the output and sending it back to the browser again via Web\+RTC for playback. \doxylink{requirements_The}{The} round-\/trip latency goes largely unnoticed from an interactivity standpoint over local wireless networks. On the client side, it\textquotesingle{}s been tested with multiple browsers including Chrome/\+\+Chromium, mobile Android, and mobile i\+OS (Safari) using H.\+264 compression.

Any application using video\+Source/\+video\+Output (including the \href{../README.md\#code-examples}{\texttt{C++/\+\+Python examples}} from this repo) can easily enable this Web\+RTC server by launching them with a stream URL of {\ttfamily webrtc:\+/\+/\+@:\+8554/\+my\+\_\+stream} or similar. Further examples are provided that build on these components and implement customizable frontends with more complex processing pipeines and web UI\textquotesingle{}s with interactive controls.\hypertarget{md_jetson-inference_2docs_2webrtc-server_autotoc_md449}{}\doxysubsection{\texorpdfstring{Enabling HTTPS /\+ SSL}{Enabling HTTPS /\+ SSL}}\label{md_jetson-inference_2docs_2webrtc-server_autotoc_md449}
It\textquotesingle{}s recommended to use secure HTTPS and SSL/\+\+TLS for transporting Web\+RTC streams and serving webpages so that they are encrypted. Also, browsers require HTTPS to use a client\textquotesingle{}s webcam from a PC. To enable HTTPS, first you need to generate a self-\/signed SSL certificate and key:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ /jetson-\/inference/data}
\DoxyCodeLine{\$\ openssl\ req\ -\/x509\ -\/newkey\ rsa:4096\ -\/keyout\ key.pem\ -\/out\ cert.pem\ -\/sha256\ -\/days\ 365\ -\/nodes\ -\/subj\ '/CN=localhost'}
\DoxyCodeLine{\$\ export\ SSL\_KEY=/jetson-\/inference/data/key.pem}
\DoxyCodeLine{\$\ export\ SSL\_CERT=/jetson-\/inference/data/cert.pem}

\end{DoxyCode}


When you set the {\ttfamily \$SSL\+\_\+\+KEY} and {\ttfamily \$SSL\+\_\+\+CERT} environment variables, they\textquotesingle{}ll automatically be picked up by applications from this repo, and HTTPS will be enabled. Otherwise, you will need to set the {\ttfamily -\/-\/ssl-\/key} and {\ttfamily -\/-\/ssl-\/cert} command-\/line arguments when you start them.

You can also store these certs anywhere you want in leui of {\ttfamily /\+jetson-\/inference/\+data} -\/ although if you\textquotesingle{}re using the Docker container, that path is recommended because it gets mounted to the host and hence your certificate will be retained after exiting the container.

When you first navigate your browser to a page that uses these self-\/signed certificates, it will issue you a warning since they don\textquotesingle{}t originate from a trusted authority:\+



You can choose to override this, and it won\textquotesingle{}t re-\/appear again until you change certificates or your device\textquotesingle{}s hostname/\+\+IP changes.\hypertarget{md_jetson-inference_2docs_2webrtc-server_autotoc_md450}{}\doxysubsection{\texorpdfstring{Sending Web\+RTC Streams}{Sending Web\+RTC Streams}}\label{md_jetson-inference_2docs_2webrtc-server_autotoc_md450}
To send any \href{aux-streaming.md\#input-streams}{\texttt{supported video stream}} to browsers over Web\+RTC for playback, simply set the \doxylink{classvideo_output}{video\+Output} URL to {\ttfamily webrtc:\+/\+/\+\texorpdfstring{$<$}{<}INTERFACE\texorpdfstring{$>$}{>}:\+\texorpdfstring{$<$}{<}PORT\texorpdfstring{$>$}{>}/\+\texorpdfstring{$<$}{<}STREAM-\/\+NAME\texorpdfstring{$>$}{>}}. Specifying {\ttfamily 0.\+0.\+0.\+0} or {\ttfamily @} as the interface will bind to all network interfaces on your device. \doxylink{requirements_The}{The} port used is typically 8554, but you can change it as you wish. \doxylink{requirements_The}{The} stream name should be unique for each video source (you can call them whatever you want, but they shouldn\textquotesingle{}t contain slashes), as these unique names allow for multiple streams to be routed by the same Web\+RTC server instance simultaneously and are used as the path to websockets.


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ video-\/viewer\ /dev/video0\ webrtc://@:8554/output\ \ \#\ stream\ V4L2\ camera\ to\ browsers\ via\ WebRTC}
\DoxyCodeLine{\$\ detectnet.py\ csi://0\ webrtc://@:8554/output\ \ \ \ \ \ \#\ stream\ MIPI\ CSI\ camera,\ with\ object\ detection}

\end{DoxyCode}


You should then be able to navigate your browser to {\ttfamily \href{https://}{\texttt{https:\+/\+/\+}}\texorpdfstring{$<$}{<}JETSON-\/\+IP\texorpdfstring{$>$}{>}:\+8554} and view the video stream like in the image above. Various connection statistics are dynamically updated on the page, like the bitrate and number of frames and packets received/\+dropped. Sometimes it can be helpful to open the browser\textquotesingle{}s debug console log ({\ttfamily Ctrl+\+Shift+I} in Chrome) as status messages about the state of the Web\+RTC connection get printed out there.\hypertarget{md_jetson-inference_2docs_2webrtc-server_autotoc_md451}{}\doxysubsubsection{\texorpdfstring{video\+Output Code}{video\+Output Code}}\label{md_jetson-inference_2docs_2webrtc-server_autotoc_md451}
