This first example is the simplest and highlights the core HTML/\+\+Java\+Script code needed to playback/\+send Web\+RTC streams and apply DNN inferencing. You can apply this to any web framework of choice should you already have a preferred frontend to integrate with.\hypertarget{md_jetson-inference_2docs_2webrtc-html_autotoc_md440}{}\doxysubsection{\texorpdfstring{Running the Example}{Running the Example}}\label{md_jetson-inference_2docs_2webrtc-html_autotoc_md440}


Launching app.\+py will start a built-\/in Python webserver (which is easy to use, but isn\textquotesingle{}t intended for production and can be easily changed out) along with an independent streaming thread that runs the Web\+RTC capture/\+transport and inferencing code:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/python/www/html}
\DoxyCodeLine{\$\ python3\ app.py\ -\/-\/classification\ \ \#\ see\ below\ for\ other\ DNN\ options}

\end{DoxyCode}


\begin{quote}
{\bfseries{note}}:\+ receiving browser webcams requires \href{webrtc-server.md\#enabling-https--ssl}{\texttt{HTTPS/\+\+SSL}} to be enabled \end{quote}


You should then be able to navigate your browser to {\ttfamily \href{https://}{\texttt{https:\+/\+/\+}}\texorpdfstring{$<$}{<}JETSON-\/\+IP\texorpdfstring{$>$}{>}:\+8050} and start the stream. 8050 is the default port used by these webapp examples, but you can change that with the {\ttfamily -\/-\/port=N} command-\/line argument. It\textquotesingle{}s also configured by default for Web\+RTC input and output, but if you want to use a different \href{aux-streaming.md\#input-streams}{\texttt{video input device}}, you can set that with the {\ttfamily -\/-\/\doxylink{group__input}{Input}} argument (for example, {\ttfamily -\/-\/\doxylink{group__input}{Input}=/\+dev/\+video0} for a V4\+L2 camera that\textquotesingle{}s directly attached to your Jetson).\hypertarget{md_jetson-inference_2docs_2webrtc-html_autotoc_md441}{}\doxysubsubsection{\texorpdfstring{Loading DNN Models}{Loading DNN Models}}\label{md_jetson-inference_2docs_2webrtc-html_autotoc_md441}
This example supports running one DNN model at a time -\/ either classification, detection, segmentation, pose estimation, action recognition, or background removal (subsequent examples support running multiple models simulateously). You can change the DNN when you launch the app like so:\+

\DoxyCodeLine{\$\ python3\ app.py\ -\/-\/detection\ -\/-\/model=ssd-\/mobilenet-\/v2}
\DoxyCodeLine{\$\ python3\ app.py\ -\/-\/segmentation\ -\/-\/model=fcn-\/resnet18-\/mhp}
\DoxyCodeLine{\$\ python3\ app.py\ -\/-\/pose\ -\/-\/model=resnet18-\/body}
\DoxyCodeLine{\$\ python3\ app.py\ -\/-\/action\ -\/-\/model=resnet18-\/kinetics}
\DoxyCodeLine{\$\ python3\ app.py\ -\/-\/background\ -\/-\/model=u2net}
