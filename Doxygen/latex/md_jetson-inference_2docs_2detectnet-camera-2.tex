\doxylink{requirements_The}{The} \href{../examples/detectnet/detectnet.cpp}{\texttt{{\ttfamily detectnet.\+cpp}}} /\+ \href{../python/examples/detectnet.py}{\texttt{{\ttfamily detectnet.\+py}}} sample that we used previously can also be used for realtime camera streaming. \doxylink{requirements_The}{The} types of supported cameras include:\+


\begin{DoxyItemize}
\item MIPI CSI cameras ({\ttfamily csi:\+/\+/\+0})
\item V4\+L2 cameras ({\ttfamily /\+dev/\+video0})
\item RTP/\+\+RTSP streams ({\ttfamily rtsp:\+/\+/\+username:\+password@ip:\+port})
\item Web\+RTC streams ({\ttfamily webrtc:\+/\+/\+@:\+port/\+stream\+\_\+name})
\end{DoxyItemize}

For more information about video streams and protocols, please see the \doxysectlink{md_jetson-inference_2docs_2aux-streaming}{Camera Streaming and Multimedia}{0} page.

Run the program with {\ttfamily -\/-\/help} to see a full list of options -\/ some of them specific to \doxylink{classdetect_net}{detect\+Net} include:\+


\begin{DoxyItemize}
\item {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag which changes the \href{detectnet-console-2.md\#pre-trained-detection-models-available}{\texttt{detection model}} being used (the default is SSD-\/\+Mobilenet-\/v2).
\item {\ttfamily -\/-\/\doxylink{group__overlay}{Overlay}} flag which can be comma-\/separated combinations of {\ttfamily box}, {\ttfamily labels}, {\ttfamily conf}, {\ttfamily track} and {\ttfamily none}
\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} default is {\ttfamily -\/-\/\doxylink{group__overlay}{Overlay}=box,labels,conf} which displays boxes, labels, and confidence values
\end{DoxyItemize}
\item {\ttfamily -\/-\/alpha} value which sets the alpha blending value used during overlay (the default is {\ttfamily 120}).
\item {\ttfamily -\/-\/threshold} value which sets the minimum threshold for detection (the default is {\ttfamily 0.\+5}).
\end{DoxyItemize}
