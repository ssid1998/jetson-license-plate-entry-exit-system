 \textsuperscript{System Setup}\hypertarget{md_jetson-inference_2docs_2jetpack-setup_autotoc_md333}{}\doxysection{\texorpdfstring{Setting up Jetson with Jet\+Pack}{Setting up Jetson with Jet\+Pack}}\label{md_jetson-inference_2docs_2jetpack-setup_autotoc_md333}
\begin{quote}
{\bfseries{note}}:\+ if your Jetson Nano has already been setup with the \href{https://developer.nvidia.com/embedded/learn/get-started-jetson-nano-devkit\#write}{\texttt{SD card image}} (which includes the Jet\+Pack components), or your Jetson has already been setup with Jet\+Pack, you can skip this step and continue to \doxysectlink{md_jetson-inference_2docs_2building-repo}{{\ttfamily Building the Repo}}{0} \end{quote}


Download the latest {\bfseries{\href{https://developer.nvidia.com/embedded/jetpack}{\texttt{Jet\+Pack}}}} to your host PC. In addition to flashing the Jetson with the latest Board Support Package (BSP), Jet\+Pack automatically installs tools for the host like CUDA Toolkit. See the Jet\+Pack \href{https://developer.nvidia.com/embedded/jetpack-notes}{\texttt{Release Notes}} for the full list of features and installed packages.

After downloading Jet\+Pack from the link above, run it from the host PC with the following commands:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash\ }
\DoxyCodeLine{\$\ cd\ <directory\ where\ you\ downloaded\ JetPack>}
\DoxyCodeLine{\$\ chmod\ +x\ JetPack-\/L4T-\/<version>-\/linux-\/x64.run\ }
\DoxyCodeLine{\$\ ./JetPack-\/L4T-\/<version>-\/linux-\/x64.run\ }

\end{DoxyCode}


\doxylink{requirements_The}{The} Jet\+Pack GUI will start. Follow the step-\/by-\/step {\bfseries{\href{http://docs.nvidia.com/jetpack-l4t/index.html\#developertools/mobile/jetpack/l4t/3.0/jetpack_l4t_install.htm}{\texttt{Install Guide}}}} to complete the setup. Near the beginning, Jet\+Pack will confirm which generation Jetson you are developing for.



Select Jetson TX1 if you are using TX1, or Jetson TX2 if you\textquotesingle{}re using TX2, and press {\ttfamily Next} to continue.

\doxylink{requirements_The}{The} next screen will list the packages available to be installed. \doxylink{requirements_The}{The} packages installed to the host are listed at the top under the {\ttfamily Host -\/ Ubuntu} dropdown, while those intended for the Jetson are shown near the bottom. You can select or deselect an individual package for installation by clicking it\textquotesingle{}s {\ttfamily Action} column.



Since CUDA will be used on the host for training DNNs, it\textquotesingle{}s recommended to select the Full install by click on the radio button in the top right. Then press {\ttfamily Next} to begin setup. Jet\+Pack will download and then install the sequence of packages. Note that all the .deb packages are stored under the {\ttfamily jetpack\+\_\+downloads} subdirectory if you are to need them later. ~\newline


After the downloads have finished installing, Jet\+Pack will enter the post-\/install phase where the Jet\+Pack is flashed with the L4T BSP. You\textquotesingle{}ll need to connect your Jetson to your host PC via the micro-\/\+USB port and cable included in the devkit. Then enter your Jetson into recovery mode by holding down the Recovery button while pressing and releasing Reset. If you type {\ttfamily lsusb} from the host PC after you\textquotesingle{}ve connected the micro-\/\+USB cable and entered the Jetson into recovery mode, you should see the NVIDIA device come up under the list of USB devices. Jet\+Pack uses the micro-\/\+USB connection from the host to flash the L4T BSP to the Jetson. ~\newline


After flashing, the Jetson will reboot and if attached to an HDMI display, will boot up to the Ubuntu desktop. After this, Jet\+Pack connects to the Jetson from the host via SSH to install additional packages to the Jetson, like the ARM aarch64 builds of CUDA Toolkit, cu\+DNN, and Tensor\+RT. For Jet\+Pack to be able to reach the Jetson via SSH, the host PC should be networked to the Jetson via Ethernet. This can be accomplished by running an Ethernet cable directly from the host to the Jetson, or by connecting both devices to a router or switch — the Jet\+Pack GUI will ask you to confirm which networking scenario is being used. ~\newline
