 \textsuperscript{Web\+App Frameworks}\hypertarget{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md443}{}\doxysection{\texorpdfstring{Recognizer (Interactive Training)}{Recognizer (Interactive Training)}}\label{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md443}
\doxylink{requirements_The}{The} Recognizer is a Flask-\/based video tagging/\+classification webapp with interactive data collection and training. As video is tagged and recorded, an updated model is incrementally re-\/trained in the background with Py\+Torch and then used for inference with Tensor\+RT. Both inference and training can run simultaneously, and the re-\/trained models are dynamically loaded at runtime for inference.



It also supports multi-\/label tagging, and in addition to recording client video over Web\+RTC, existing images can be uploaded from the client. \doxylink{requirements_The}{The} main source files for this example (found under \href{../python/www/recognizer}{\texttt{{\ttfamily python/\+www/\+recognizer}}}) are as follows:\+


\begin{DoxyItemize}
\item \href{../python/www/recognizer/app.py}{\texttt{{\ttfamily app.\+py}}} (webserver)
\item \href{../python/www/recognizer/stream.py}{\texttt{{\ttfamily stream.\+py}}} (Web\+RTC streaming thread)
\item \href{../python/www/recognizer/model.py}{\texttt{{\ttfamily model.\+py}}} (DNN inferencing + training)
\item \href{../python/www/recognizer/dataset.py}{\texttt{{\ttfamily dataset.\+py}}} (data tagging + recording)
\item \href{../python/www/recognizer/templates/index.html}{\texttt{{\ttfamily index.\+html}}} (frontend presentation)
\end{DoxyItemize}\hypertarget{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md444}{}\doxysubsection{\texorpdfstring{Running the Example}{Running the Example}}\label{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md444}

\doxylink{requirements_The}{The} {\ttfamily -\/-\/data} argument sets the path where your dataset and models are stored under. If you built jetson-\/inference from source, you should elect to \href{building-repo-2.md\#installing-pytorch}{\texttt{Install Py\+Torch}} (or run the {\ttfamily install-\/pytorch.\+sh} script again). If you\textquotesingle{}re using the Docker container, Py\+Torch is already installed for you.

After running app.\+py, you should be able to navigate your browser to {\ttfamily \href{https://}{\texttt{https:\+/\+/\+}}\texorpdfstring{$<$}{<}JETSON-\/\+IP\texorpdfstring{$>$}{>}:\+8050} and start the stream. \doxylink{requirements_The}{The} default port is 8050, but you can change that with the {\ttfamily -\/-\/port=N} command-\/line argument. It\textquotesingle{}s configured by default for Web\+RTC input and output, but if you want to use a different \href{aux-streaming.md\#input-streams}{\texttt{video input device}}, you can set that with the {\ttfamily -\/-\/\doxylink{group__input}{Input}} argument (for example, {\ttfamily -\/-\/\doxylink{group__input}{Input}=/\+dev/\+video0} for a V4\+L2 camera)\hypertarget{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md445}{}\doxysubsubsection{\texorpdfstring{Collecting Data}{Collecting Data}}\label{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md445}
If needed first select a client camera from the stream source dropdown on the webpage, and press the {\ttfamily Send} button. When ready, enter class tag(s) of what the camera is looking at in the Tags selection box. Once a tag is entered, you\textquotesingle{}ll be able to either Record or Upload images into the dataset. You can hold down the Record button to capture a video sequence. Below is a high-\/level diagram of the data flow:\+



It\textquotesingle{}s recommended to keep the distribution of tags across the classes relatively balanced -\/ otherwise the model will be more likely to be biased towards certain classes. You can view the label distribution and number of images in the dataset by expanding the {\ttfamily Training} dropdown.\hypertarget{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md446}{}\doxysubsubsection{\texorpdfstring{Training}{Training}}\label{md_jetson-inference_2docs_2webrtc-recognizer_autotoc_md446}
As you add and tag new data, training can be enabled under the {\ttfamily Training} dropdown. \doxylink{requirements_The}{The} training progress and accuracy will be updated on the page. At the end of each epoch, if the model has the highest accuracy it will be exported to ONNX and loaded into Tensor\+RT for inference.
