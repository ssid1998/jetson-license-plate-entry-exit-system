 \textsuperscript{Mono Depth}\hypertarget{md_jetson-inference_2docs_2depthnet_autotoc_md165}{}\doxysection{\texorpdfstring{Monocular Depth with Depth\+Net}{Monocular Depth with Depth\+Net}}\label{md_jetson-inference_2docs_2depthnet_autotoc_md165}
Depth sensing is useful for tasks such as mapping, navigation and obstacle detection, however it historically required a stereo camera or RGB-\/D camera. There are now DNNs that are able to infer relative depth from a single monocular image (aka mono depth). See the \href{https://arxiv.org/abs/1903.03273}{\texttt{MIT Fast\+Depth}} paper for one such approach to accomplishing this using Fully-\/\+Convolutional Networks (FCNs).



\doxylink{requirements_The}{The} \href{../c/depthNet.h}{\texttt{{\ttfamily depth\+Net}}} object accepts a single color image as input, and outputs the depth map. \doxylink{requirements_The}{The} depth map is colorized for visualization, but the raw depth field is also accessible for directly accessing the depths. \href{../c/depthNet.h}{\texttt{{\ttfamily depth\+Net}}} is available to use from \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.inference.html\#depthNet}{\texttt{Python}} and \href{../c/depthNet.h}{\texttt{C++}}.

As examples of using the {\ttfamily \doxylink{classdepth_net}{depth\+Net}} class, we provide sample programs for C++ and Python:\+


\begin{DoxyItemize}
\item \href{../examples/depthnet/depthnet.cpp}{\texttt{{\ttfamily depthnet.\+cpp}}} (C++)
\item \href{../python/examples/depthnet.py}{\texttt{{\ttfamily depthnet.\+py}}} (Python)
\end{DoxyItemize}

These samples are able to infer depth from images, videos, and camera feeds. For more info about the various types of input/\+output streams supported, see the \doxysectlink{md_jetson-inference_2docs_2aux-streaming}{Camera Streaming and Multimedia}{0} page.\hypertarget{md_jetson-inference_2docs_2depthnet_autotoc_md166}{}\doxysubsection{\texorpdfstring{Mono Depth on Images}{Mono Depth on Images}}\label{md_jetson-inference_2docs_2depthnet_autotoc_md166}
First, let\textquotesingle{}s try running the {\ttfamily depthnet} sample on some examples images. In addition to the input/\+output paths, there are some additional command-\/line options that are optional:\+


\begin{DoxyItemize}
\item optional {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag which changes the depth model being used (recommended default is {\ttfamily fcn-\/mobilenet}).
\item optional {\ttfamily -\/-\/visualize} flag which can be comma-\/separated combinations of {\ttfamily \doxylink{group__input}{Input}}, {\ttfamily depth}
\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} default is {\ttfamily -\/-\/visualize=\doxylink{group__input}{Input},depth} which displays the input and depth images side-\/by-\/side
\item To view only the depth image, use {\ttfamily -\/-\/visualize=depth}
\end{DoxyItemize}
\item optional {\ttfamily -\/-\/depth-\/size} value which scales the size of the depth map relative to the input (the default is {\ttfamily 1.\+0})
\item optional {\ttfamily -\/-\/filter-\/mode} flag which selects {\ttfamily point} or {\ttfamily linear} filtering used for upsampling (the default is {\ttfamily linear})
\item optional {\ttfamily -\/-\/\doxylink{group__colormap}{Color Mapping}} flag which sets the color mapping to use during visualization (the default is {\ttfamily viridis\+\_\+inverted})
\end{DoxyItemize}

If you\textquotesingle{}re using the \doxysectlink{md_jetson-inference_2docs_2aux-docker}{Docker container}{0}, it\textquotesingle{}s recommended to save the output images to the {\ttfamily images/\+test} mounted directory. These images will then be easily viewable from your host device under {\ttfamily jetson-\/inference/\+data/\+images/\+test} (for more info, see \href{aux-docker.md\#mounted-data-volumes}{\texttt{Mounted Data Volumes}}).

\hypertarget{md_jetson-inference_2docs_2depthnet_autotoc_md168}{}\doxysubsection{\texorpdfstring{Getting the Raw Depth Field}{Getting the Raw Depth Field}}\label{md_jetson-inference_2docs_2depthnet_autotoc_md168}
If you want to access the raw depth map, you can do so with {\ttfamily \doxylink{classdepth_net_a2f1ea14537fb97505adb250db0201950}{depth\+Net.\+Get\+Depth\+Field()}}. This will return a single-\/channel floating point image that is typically smaller (224x224) than the original input -\/ this represents the raw output of the model. On the other hand, the colorized depth image used for visualization is upsampled to match the resolution of the original input (or whatever the {\ttfamily -\/-\/depth-\/size} scale is set to).

Trying to measure absolute distances using mono depth can lead to inaccuracies as it\textquotesingle{}s typically more effective at relative depth estimation. \doxylink{requirements_The}{The} range of values in the raw depth field can vary depending on the scene, so these are often re-\/calculated dynamically. For example, during visualization histogram equalization is performed on the depth field to distribute the colormap more evenly across the range of depth values.
