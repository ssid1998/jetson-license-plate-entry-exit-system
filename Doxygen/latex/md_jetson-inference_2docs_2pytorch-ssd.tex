 \textsuperscript{Transfer Learning -\/ Object Detection}\hypertarget{md_jetson-inference_2docs_2pytorch-ssd_autotoc_md376}{}\doxysection{\texorpdfstring{Re-\/training SSD-\/\+Mobilenet}{Re-\/training SSD-\/\+Mobilenet}}\label{md_jetson-inference_2docs_2pytorch-ssd_autotoc_md376}
Next, we\textquotesingle{}ll train our own SSD-\/\+Mobilenet object detection model using Py\+Torch and the \href{https://storage.googleapis.com/openimages/web/visualizer/index.html?set=train&type=detection&c=\%2Fm\%2F06l9r}{\texttt{Open Images}} dataset. SSD-\/\+Mobilenet is a popular network architecture for realtime object detection on mobile and embedded devices that combines the \href{https://arxiv.org/abs/1512.02325}{\texttt{SSD-\/300}} Single-\/\+Shot Multi\+Box Detector with a \href{https://arxiv.org/abs/1704.04861}{\texttt{Mobilenet}} backbone. ~\newline


\href{https://arxiv.org/abs/1512.02325}{\texttt{}}

In the example below, we\textquotesingle{}ll train a custom detection model that locates 8 different varieties of fruit, although you are welcome to pick from any of the \href{https://github.com/dusty-nv/pytorch-ssd/blob/master/open_images_classes.txt}{\texttt{600 classes}} in the Open Images dataset to train your model on. You can visually browse the dataset \href{https://storage.googleapis.com/openimages/web/visualizer/index.html?set=train&type=detection}{\texttt{here}}.



To get started, first make sure that you have \href{https://developer.nvidia.com/embedded/jetpack}{\texttt{Jet\+Pack 4.\+4}} (or newer) and \href{pytorch-transfer-learning.md\#installing-pytorch}{\texttt{Py\+Torch installed}} for {\bfseries{Python 3}} on your Jetson. Jet\+Pack 4.\+4 includes Tensor\+RT 7.\+1, which is the minimum Tensor\+RT version that supports loading SSD-\/\+Mobilenet via ONNX. Newer versions of Tensor\+RT are fine too.\hypertarget{md_jetson-inference_2docs_2pytorch-ssd_autotoc_md377}{}\doxysubsection{\texorpdfstring{Setup}{Setup}}\label{md_jetson-inference_2docs_2pytorch-ssd_autotoc_md377}

In practice, to keep the training time down (and disk space), you probably want to keep the total number of images \texorpdfstring{$<$}{<}10K. Although the more images you use, the more accurate your model will be. You can limit the amount of data downloaded with the {\ttfamily -\/-\/max-\/images} option or the {\ttfamily -\/-\/max-\/annotations-\/per-\/class} options:\+


\begin{DoxyItemize}
\item {\ttfamily -\/-\/max-\/images} limits the total dataset to the specified number of images, while keeping the distribution of images per class roughly the same as the original dataset. If one class has more images than another, the ratio will remain roughly the same.
\item {\ttfamily -\/-\/max-\/annotations-\/per-\/class} limits each class to the specified number of bounding boxes, and if a class has less than that number available, all of it\textquotesingle{}s data will be used -\/ this is useful if the distribution of data is unbalanced across classes.
\end{DoxyItemize}

For example, if you wanted to only use 2500 images for the fruit dataset, you would launch the downloader like this:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ python3\ open\_images\_downloader.py\ -\/-\/max-\/images=2500\ -\/-\/class-\/names\ "{}Apple,Orange,Banana,Strawberry,Grape,Pear,Pineapple,Watermelon"{}\ -\/-\/data=data/fruit}

\end{DoxyCode}


If the {\ttfamily -\/-\/max-\/boxes} option or {\ttfamily -\/-\/max-\/annotations-\/per-\/class} isn\textquotesingle{}t set, by default all the data available will be downloaded -\/ so beforehand, be sure to check the amount of data first with {\ttfamily -\/-\/stats-\/only}. Unfortunately it isn\textquotesingle{}t possible in advance to determine the actual disk size requirements of the images, but a general rule of thumb for this dataset is to budget \texorpdfstring{$\sim$}{\string~}350\+KB per image (\texorpdfstring{$\sim$}{\string~}2\+GB for the fruits).\hypertarget{md_jetson-inference_2docs_2pytorch-ssd_autotoc_md380}{}\doxysubsubsection{\texorpdfstring{Training Performance}{Training Performance}}\label{md_jetson-inference_2docs_2pytorch-ssd_autotoc_md380}
