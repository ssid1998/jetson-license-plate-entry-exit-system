This project supports capturing and streaming video feeds \& static images via a variety of interfaces and protocols, including:\+


\begin{DoxyItemize}
\item MIPI CSI cameras
\item V4\+L2 cameras
\item Web\+RTC
\item RTP /\+ RTSP
\item Videos \& Images
\item Image sequences
\item Open\+GL windows
\end{DoxyItemize}

Streams are identified via a resource \doxylink{struct_u_r_i}{URI} and are acquired through the \`{}video\+Source\`{} and \`{}video\+Output\`{} APIs. These settings can be configured via command-\/line arguments or in your application\textquotesingle{}s source code. \doxylink{requirements_The}{The} tables below show the supported input/\+output protocols and \doxylink{struct_u_r_i}{URI} formats:\+\hypertarget{md_jetson-inference_2docs_2aux-streaming_autotoc_md94}{}\doxysubsubsection{\texorpdfstring{Input Streams}{Input Streams}}\label{md_jetson-inference_2docs_2aux-streaming_autotoc_md94}

\hypertarget{md_jetson-inference_2docs_2aux-streaming_autotoc_md101}{}\doxysubsubsubsection{\texorpdfstring{V4\+L2 Formats}{V4\+L2 Formats}}\label{md_jetson-inference_2docs_2aux-streaming_autotoc_md101}
By default, V4\+L2 cameras will be created using the camera format with the highest framerate that most closely matches the desired resolution (by default, that resolution is 1280x720). \doxylink{requirements_The}{The} format with the highest framerate may be encoded (for example with H.\+264 or MJPEG), as USB cameras typically transmit uncompressed YUV/\+\+RGB at lower framerates. In this case, that codec will be detected and the camera stream will automatically be decoded using the Jetson\textquotesingle{}s hardware decoder to attain the highest framerate.

If you explicitly want to choose the format used by the V4\+L2 camera, you can do so with the {\ttfamily -\/-\/input-\/width}, {\ttfamily -\/-\/input-\/height}, and {\ttfamily -\/-\/input-\/codec} options. Possible decoder codec options are {\ttfamily -\/-\/input-\/codec=h264, h265, vp8, vp9, mpeg2, mpeg4, mjpeg}


\begin{DoxyCode}{0}
\DoxyCodeLine{\$\ video-\/viewer\ -\/-\/input-\/width=1920\ -\/-\/input-\/height=1080\ -\/-\/input-\/codec=h264\ /dev/video0}

\end{DoxyCode}

This projects includes a built-\/in Web\+RTC server (input/\+output) for streaming video to/\+from client web browsers. You can use this for conveniently viewing video streams when your Jetson is headless and doesn\textquotesingle{}t have a display attached, or for easily building interactive webapps that use Jetson and edge AI on the backend. Tested browsers include Chrome/\+\+Chromium, mobile Android, and mobile i\+OS (Safari).
