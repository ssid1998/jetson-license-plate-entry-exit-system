\doxylink{requirements_The}{The} loaded image will be stored in shared memory that\textquotesingle{}s mapped to both the CPU and GPU. There are two pointers available for access in the CPU and GPU address spaces, but there is really only one copy of the image in memory. Both the CPU and GPU pointers resolve to the same physical memory, without needing to perform memory copies (i.\+e. {\ttfamily cuda\+Memcpy()}). ~\newline


When accessing the image from CPU code, the {\ttfamily img\+CPU} pointer should be used, and when accessing the image from within a CUDA kernel on the GPU, the {\ttfamily img\+CUDA} pointer should be used. Since our operations in this example will run on the GPU with Tensor\+RT, we will be using the {\ttfamily img\+CUDA} pointer. ~\newline


\doxylink{requirements_The}{The} image is loaded in {\ttfamily float4} RGBA format, with pixel values between 0.\+0 and 255.\+0. ~\newline
