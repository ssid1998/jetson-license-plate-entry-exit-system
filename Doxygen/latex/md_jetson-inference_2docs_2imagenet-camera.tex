Next we have a realtime image recognition camera demo available for C++ and Python:\+


\begin{DoxyItemize}
\item \href{../examples/imagenet-camera/imagenet-camera.cpp}{\texttt{{\ttfamily imagenet-\/camera.\+cpp}}} (C++)
\item \href{../python/examples/imagenet-camera.py}{\texttt{{\ttfamily imagenet-\/camera.\+py}}} (Python)
\end{DoxyItemize}

Similar to the previous \doxysectlink{md_jetson-inference_2docs_2imagenet-console}{{\ttfamily imagenet-\/console}}{0} example, the camera applications are built to the {\ttfamily /\+aarch64/\+bin} directory. They run on a live camera stream with Open\+GL rendering and accept 4 optional command-\/line arguments:\+


\begin{DoxyItemize}
\item {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag setting the classification model (default is Google\+Net)
\begin{DoxyItemize}
\item See \href{imagenet-console.md\#downloading-other-classification-models}{\texttt{Downloading Other Classification Models}} for the networks available to use.
\end{DoxyItemize}
\item {\ttfamily -\/-\/\doxylink{group__camera}{Camera Capture}} flag setting the camera device to use
\begin{DoxyItemize}
\item MIPI CSI cameras are used by specifying the sensor index ({\ttfamily 0} or {\ttfamily 1}, ect.)
\item V4\+L2 USB cameras are used by specifying their {\ttfamily /\+dev/\+video} node ({\ttfamily /\+dev/\+video0}, {\ttfamily /\+dev/\+video1}, ect.)
\item \doxylink{requirements_The}{The} default is to use MIPI CSI sensor 0 ({\ttfamily -\/-\/\doxylink{group__camera}{Camera Capture}=0})
\end{DoxyItemize}
\item {\ttfamily -\/-\/width} and {\ttfamily -\/-\/height} flags setting the camera resolution (default is {\ttfamily 1280x720})
\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} resolution should be set to a format that the camera supports.
\begin{DoxyItemize}
\item Query the available formats with the following commands:\+ ~\newline
 {\ttfamily  bash \$ sudo apt-\/get install v4l-\/utils \$ v4l2-\/ctl -\/-\/list-\/formats-\/ext }
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}

You can combine the usage of these flags as needed, and there are additional command line parameters available for loading custom models. Launch the application with the {\ttfamily -\/-\/help} flag to recieve more info, or see the \href{../README.md\#code-examples}{\texttt{{\ttfamily Examples}}} readme.

Displayed in the Open\+GL window are the live camera stream, the classified object name, and the confidence of the classified object, along with the framerate of the network. On Jetson Nano you should see up to around \texorpdfstring{$\sim$}{\string~}75 FPS for Google\+Net and Res\+Net-\/18 (faster on other Jetson\textquotesingle{}s).

  

\doxylink{requirements_The}{The} application can recognize up to 1000 different types of objects, since the classification models are trained on the ILSVRC Image\+Net dataset which contains 1000 classes of objects. \doxylink{requirements_The}{The} mapping of names for the 1000 types of objects, you can find in the repo under \href{http://github.com/dusty-nv/jetson-inference/blob/master/data/networks/ilsvrc12_synset_words.txt}{\texttt{{\ttfamily data/\+networks/\+ilsvrc12\+\_\+synset\+\_\+words.\+txt}}}
