 \textsuperscript{Image Classification}

\hypertarget{md_jetson-inference_2docs_2imagenet-console-2_autotoc_md259}{}\doxysection{\texorpdfstring{Classifying Images with Image\+Net}{Classifying Images with Image\+Net}}\label{md_jetson-inference_2docs_2imagenet-console-2_autotoc_md259}
There are multiple types of deep learning networks available, including recognition, detection/\+localization, and semantic segmentation. \doxylink{requirements_The}{The} first deep learning capability we\textquotesingle{}re highlighting in this tutorial is {\bfseries{image recognition}}, using classifcation networks that have been trained on large datasets to identify scenes and objects.



\doxylink{requirements_The}{The} \href{../c/imageNet.h}{\texttt{{\ttfamily image\+Net}}} object accepts an input image and outputs the probability for each class. Having been trained on the Image\+Net ILSVRC dataset of {\bfseries{\href{../data/networks/ilsvrc12_synset_words.txt}{\texttt{1000 objects}}}}, the Google\+Net and Res\+Net-\/18 models were automatically downloaded during the build step. See below for other classification models that can be downloaded and used as well.

As an example of using the \href{../c/imageNet.h}{\texttt{{\ttfamily image\+Net}}} class, we provide sample programs for C++ and Python:\+


\begin{DoxyItemize}
\item \href{../examples/imagenet/imagenet.cpp}{\texttt{{\ttfamily imagenet.\+cpp}}} (C++)
\item \href{../python/examples/imagenet.py}{\texttt{{\ttfamily imagenet.\+py}}} (Python)
\end{DoxyItemize}

These samples are able to classify images, videos, and camera feeds. For more info about the various types of input/\+output streams supported, see the \doxysectlink{md_jetson-inference_2docs_2aux-streaming}{Camera Streaming and Multimedia}{0} page.\hypertarget{md_jetson-inference_2docs_2imagenet-console-2_autotoc_md260}{}\doxysubsubsection{\texorpdfstring{Using the Image\+Net Program on Jetson}{Using the Image\+Net Program on Jetson}}\label{md_jetson-inference_2docs_2imagenet-console-2_autotoc_md260}
First, let\textquotesingle{}s try using the {\ttfamily imagenet} program to test \doxylink{classimage_net}{image\+Net} recognition on some example images. It loads an image (or images), uses Tensor\+RT and the {\ttfamily \doxylink{classimage_net}{image\+Net}} class to perform the inference, then overlays the classification result and saves the output image. \doxylink{requirements_The}{The} project comes with sample images for you to use located under the {\ttfamily images/\+} directory.

After \doxysectlink{md_jetson-inference_2docs_2building-repo-2}{building}{0} the project, make sure your terminal is located in the {\ttfamily aarch64/\+bin} directory:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/build/aarch64/bin}

\end{DoxyCode}


Next, let\textquotesingle{}s classify an example image with the {\ttfamily imagenet} program, using either the \href{../examples/imagenet/imagenet.cpp}{\texttt{C++}} or \href{../python/examples/imagenet.py}{\texttt{Python}} variants. If you\textquotesingle{}re using the \doxysectlink{md_jetson-inference_2docs_2aux-docker}{Docker container}{0}, it\textquotesingle{}s recommended to save the classified output image to the {\ttfamily images/\+test} mounted directory. These images will then be easily viewable from your host device in the {\ttfamily jetson-\/inference/\+data/\+images/\+test} directory (for more info, see \href{aux-docker.md\#mounted-data-volumes}{\texttt{Mounted Data Volumes}}). ~\newline

Generally the more complex networks can have greater classification accuracy, with increased runtime.\hypertarget{md_jetson-inference_2docs_2imagenet-console-2_autotoc_md262}{}\doxysubsubsection{\texorpdfstring{Using Different Classification Models}{Using Different Classification Models}}\label{md_jetson-inference_2docs_2imagenet-console-2_autotoc_md262}
You can specify which model to load by setting the {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag on the command line to one of the corresponding CLI arguments from the table above. By default, Google\+Net is loaded if the optional {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag isn\textquotesingle{}t specified.
