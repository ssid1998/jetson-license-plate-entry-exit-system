 \textsuperscript{Web\+App Frameworks}\hypertarget{md_jetson-inference_2docs_2webrtc-dash_autotoc_md426}{}\doxysection{\texorpdfstring{Plotly Dashboard}{Plotly Dashboard}}\label{md_jetson-inference_2docs_2webrtc-dash_autotoc_md426}
\href{https://plotly.com/dash/}{\texttt{Plotly Dash}} is a Python-\/based web framework for building data-\/driven dashboards and interactive UI\textquotesingle{}s. On the frontend it uses \href{https://reactjs.org/}{\texttt{React.\+js}} client-\/side, which connects state changes to Python callbacks running on the server. With it, you can quickly develop rich visualizations that integrate with backend processing pipelines and data analytics. In this example (found under \href{../python/www/dash}{\texttt{{\ttfamily python/\+www/\+dash}}}), users can dynamically create streams, load DNN models, visualize events, and setup extendable actions triggered by events:\+



As before, it uses Web\+RTC for streaming live video and Tensor\+RT for inferencing. Note that this sample is still under development as a proof-\/of-\/concept. For reference, the project is structured as follows:\+

\item \href{../python/www/dash/layout}{\texttt{{\ttfamily layout/\+}}} (UI components)
\item \href{../python/www/dash/server}{\texttt{{\ttfamily server/\+}}} (backend streaming/\+processing)
\end{DoxyItemize}\hypertarget{md_jetson-inference_2docs_2webrtc-dash_autotoc_md427}{}\doxysubsection{\texorpdfstring{Running the Example}{Running the Example}}\label{md_jetson-inference_2docs_2webrtc-dash_autotoc_md427}
Launching app.\+py will start the dashboard, along with a backend process that runs the Web\+RTC capture/\+transport, inferencing, and event/\+action triggers. Whereas previous examples ran the streaming in a thread inside same process as the webserver, running the streaming backend in it\textquotesingle{}s own independent process allows for multiple webserver workers to load-\/balance in deployment (i.\+e. with \href{https://gunicorn.org/}{\texttt{Gunicorn}} or other production WSGI webservers). These processes share metadata over REST JSON messages (with the video data residing within the streaming process).

When the output of a DNN changes (i.\+e. the classification result changes, or a new object is detected) it logs an event in the system. These events can be monitored in realtime by opening the Events Table from under the {\ttfamily Events} menu:\+



You can filter and sort by column in the table, and visualize the results in the \doxylink{class_event}{Event} Timeline:\+



\doxylink{requirements_The}{The} y-\/axis of this plot shows confidence score, the x-\/axis shows time, and each object class gets a different trace in the chart. Quickly creating different types of dynamic \href{https://plotly.com/python/}{\texttt{graphs}} and \href{https://dash.plotly.com/datatable}{\texttt{tables}} is a strong feature of Plotly Dash, and you can extend these when creating your own apps.\hypertarget{md_jetson-inference_2docs_2webrtc-dash_autotoc_md432}{}\doxysubsection{\texorpdfstring{Actions}{Actions}}\label{md_jetson-inference_2docs_2webrtc-dash_autotoc_md432}
Actions are plugins that filter events and trigger user-\/defined code (such as alerts/\+notifications, playing sounds, or generating a physical response) when such an event occurs. An action\textquotesingle{}s properties are exposed through the web UI so that they are configurable at runtime:\+



You can add your own action types under the project\textquotesingle{}s \href{../python/www/dash/actions}{\texttt{{\ttfamily actions/\+}}} directory, and they\textquotesingle{}ll automatically be loaded by the app at start-\/up and selectable from the UI. Multiple instances of a type of action can be created by the user, each with independent settings they can control.

For example, here\textquotesingle{}s the template for an action that simply logs messages to the server\textquotesingle{}s terminal:\+

Action plugins should implement the {\ttfamily on\+\_\+event()} callback, which receives all new and updated events from the system. \doxylink{requirements_The}{The} plugins then filter the events by domain-\/specific critera before triggering a response of some kind. See the \href{../python/www/dash/server/event.py}{\texttt{{\ttfamily Event}}} class for the event attributes that can be accessed. This code all runs in the backend streaming process, and has access to the low-\/level data streams without impacting performance.\hypertarget{md_jetson-inference_2docs_2webrtc-dash_autotoc_md433}{}\doxysubsubsection{\texorpdfstring{Filters}{Filters}}\label{md_jetson-inference_2docs_2webrtc-dash_autotoc_md433}
