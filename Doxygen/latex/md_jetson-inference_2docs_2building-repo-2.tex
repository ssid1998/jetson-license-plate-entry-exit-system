\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md136}{}\doxysection{\texorpdfstring{Building the Project from Source}{Building the Project from Source}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md136}
Once your Jetson has has been \doxysectlink{md_jetson-inference_2docs_2jetpack-setup-2}{flashed with Jet\+Pack}{0} or setup with the latest \href{https://developer.nvidia.com/embedded/learn/get-started-jetson-nano-devkit\#write}{\texttt{SD card image}}, you should be able to either run the pre-\/built \doxysectlink{md_jetson-inference_2docs_2aux-docker}{Docker container}{0}, or compile jetson-\/inference from source. Either option is also able to install Py\+Torch for you.\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md137}{}\doxysubsection{\texorpdfstring{Quick Reference}{Quick Reference}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md137}
If you aren\textquotesingle{}t using the \doxysectlink{md_jetson-inference_2docs_2aux-docker}{Docker container}{0}, here\textquotesingle{}s a condensed form of the commands to build/\+install the project directly on your Jetson:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{sudo\ apt-\/get\ update}
\DoxyCodeLine{sudo\ apt-\/get\ install\ git\ cmake\ libpython3-\/dev\ python3-\/numpy}
\DoxyCodeLine{git\ clone\ -\/-\/recursive\ -\/-\/depth=1\ https://github.com/dusty-\/nv/jetson-\/inference}
\DoxyCodeLine{cd\ jetson-\/inference}
\DoxyCodeLine{mkdir\ build}
\DoxyCodeLine{cd\ build}
\DoxyCodeLine{cmake\ ../}
\DoxyCodeLine{make\ -\/j\$(nproc)}
\DoxyCodeLine{sudo\ make\ install}
\DoxyCodeLine{sudo\ ldconfig}

\end{DoxyCode}
 Below we will go through each step and discuss various build options along the way.\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md138}{}\doxysubsection{\texorpdfstring{Cloning the Repo}{Cloning the Repo}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md138}
To download the code, navigate to a folder of your choosing on the Jetson. First, make sure git and cmake are installed:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ sudo\ apt-\/get\ update}
\DoxyCodeLine{\$\ sudo\ apt-\/get\ install\ git\ cmake}

\end{DoxyCode}


Then clone the {\ttfamily jetson-\/inference} project:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ git\ clone\ https://github.com/dusty-\/nv/jetson-\/inference}
\DoxyCodeLine{\$\ cd\ jetson-\/inference}
\DoxyCodeLine{\$\ git\ submodule\ update\ -\/-\/init}

\end{DoxyCode}


Remember to run the {\ttfamily git submodule update -\/-\/init} step (or clone with the {\ttfamily -\/-\/recursive} flag).\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md139}{}\doxysubsubsection{\texorpdfstring{Python Development Packages}{Python Development Packages}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md139}
\doxylink{requirements_The}{The} Python functionality of this project is implemented through Python extension modules that provide bindings to the native C++ code using the Python C API. While configuring the project, the repo searches for versions of Python that have development packages installed on the system, and will then build the bindings for each version of Python that\textquotesingle{}s present (e.\+g. Python 2.\+7, 3.\+6, 3.\+8).


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ sudo\ apt-\/get\ install\ libpython3-\/dev\ python3-\/numpy}

\end{DoxyCode}


Then after the {\ttfamily sudo make install} step, the \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.inference.html}{\texttt{{\ttfamily jetson\+\_\+inference}}} and \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.utils.html}{\texttt{{\ttfamily jetson\+\_\+utils}}} modules should be able to be imported in Python.\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md140}{}\doxysubsubsection{\texorpdfstring{Configuring with CMake}{Configuring with CMake}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md140}
Next, create a build directory within the project and run {\ttfamily cmake} to configure the build. When {\ttfamily cmake} is run, a script is launched (\href{../CMakePreBuild.sh}{\texttt{{\ttfamily CMake\+Pre\+Build.\+sh}}}) that will install any required dependencies and download DNN models for you.


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference\ \ \ \ \#\ omit\ if\ working\ directory\ is\ already\ jetson-\/inference/\ from\ above}
\DoxyCodeLine{\$\ mkdir\ build}
\DoxyCodeLine{\$\ cd\ build}
\DoxyCodeLine{\$\ cmake\ ../}

\end{DoxyCode}


\begin{quote}
{\bfseries{note}}:\+ this command will launch the \href{../CMakePreBuild.sh}{\texttt{{\ttfamily CMake\+Pre\+Build.\+sh}}} script which asks for sudo privileges while installing some prerequisite packages on the Jetson. \doxylink{requirements_The}{The} script also downloads pre-\/trained networks from web services. \end{quote}
\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md141}{}\doxysubsection{\texorpdfstring{Installing Py\+Torch}{Installing Py\+Torch}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md141}
If you are using Jet\+Pack 4.\+2 or newer, another tool will now run that can optionally install Py\+Torch on your Jetson if you want to re-\/train networks with \doxysectlink{md_jetson-inference_2docs_2pytorch-transfer-learning}{transfer learning}{0} later in the tutorial. This step is optional, and if you don\textquotesingle{}t wish to do the transfer learning steps, you don\textquotesingle{}t need to install Py\+Torch and can skip this step.

If desired, select the Py\+Torch package versions for Python 2.\+7 and/\+or Python3 that you want installed and hit {\ttfamily Enter} to continue. Otherwise, leave the options un-\/selected, and it will skip the installation of Py\+Torch.



\begin{quote}
{\bfseries{note}}:\+ the automated Py\+Torch installation tool requires Jet\+Pack 4.\+2 (or newer)~\newline
 ~~~~~~~~~~for other versions, see \href{https://elinux.org/Jetson_Zoo\#PyTorch_.28Caffe2.29}{\texttt{{\ttfamily http:\+/\+/\+e\+Linux.\+org/\+\+Jetson\+\_\+\+Zoo}}} to build from source. \end{quote}


You can also run this tool again later if you decide that you want to install Py\+Torch at another time:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/build}
\DoxyCodeLine{\$\ ./install-\/pytorch.sh}

\end{DoxyCode}


Running these commands will prompt you with the same dialog as seen above.\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md142}{}\doxysubsection{\texorpdfstring{Compiling the Project}{Compiling the Project}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md142}
Make sure you are still in the {\ttfamily jetson-\/inference/\+build} directory, created above in step \#3.

Then run {\ttfamily make} followed by {\ttfamily sudo make install} to build the libraries, Python extension bindings, and code samples:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/build\ \ \ \ \ \ \ \ \ \ \#\ omit\ if\ working\ directory\ is\ already\ build/\ from\ above}
\DoxyCodeLine{\$\ make}
\DoxyCodeLine{\$\ sudo\ make\ install}
\DoxyCodeLine{\$\ sudo\ ldconfig}

\end{DoxyCode}


\doxylink{requirements_The}{The} project will be built to {\ttfamily jetson-\/inference/\+build/\+aarch64}, with the following directory structure:\+

In the build tree, you can find the binaries residing in {\ttfamily build/\+aarch64/\+bin/\+}, headers in {\ttfamily build/\+aarch64/\+include/\+}, and libraries in {\ttfamily build/\+aarch64/\+lib/\+}. These also get installed under {\ttfamily /\+usr/\+local/\+} during the {\ttfamily sudo make install} step.

\doxylink{requirements_The}{The} Python bindings for the \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.inference.html}{\texttt{{\ttfamily jetson.\+inference}}} and \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.utils.html}{\texttt{{\ttfamily jetson.\+utils}}} modules also get installed during the {\ttfamily sudo make install} step under {\ttfamily /\+usr/\+lib/\+python\texorpdfstring{$\ast$}{*}/\+dist-\/packages/\+}. If you update the code, remember to run it again.\hypertarget{md_jetson-inference_2docs_2building-repo-2_autotoc_md143}{}\doxysubsection{\texorpdfstring{Digging Into the Code}{Digging Into the Code}}\label{md_jetson-inference_2docs_2building-repo-2_autotoc_md143}
See the {\bfseries{\href{../README.md\#api-reference}{\texttt{API Reference}}}} documentation for the vision primitives available in {\ttfamily libjetson-\/inference}, including {\ttfamily \doxylink{classimage_net}{image\+Net}} for image recognition, {\ttfamily \doxylink{classdetect_net}{detect\+Net}} for object localization, and {\ttfamily \doxylink{classseg_net}{seg\+Net}} for semantic segmentation.
