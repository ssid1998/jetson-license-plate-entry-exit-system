 \textsuperscript{Object Detection}\hypertarget{md_jetson-inference_2docs_2detectnet-console_autotoc_md190}{}\doxysection{\texorpdfstring{Detecting Objects from the Command Line}{Detecting Objects from the Command Line}}\label{md_jetson-inference_2docs_2detectnet-console_autotoc_md190}
To process test images with \href{../c/detectNet.h}{\texttt{{\ttfamily detect\+Net}}} and Tensor\+RT on the Jetson, we can use the \href{../detectnet-console/detectnet-console.cpp}{\texttt{{\ttfamily detectnet-\/console}}} program. ~\newline


\href{../detectnet-console/detectnet-console.cpp}{\texttt{{\ttfamily detectnet-\/console}}} accepts command-\/line arguments representing the path to the input image and path to the output image (with the bounding box overlays rendered). Some test images are also included with the repo.

To specify your model that you downloaded from DIGITS in the previous step, use the syntax to {\ttfamily detectnet-\/console} below. First, for convienience, set the path to your extracted snapshot into a {\ttfamily \$NET} variable:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ NET=20170504-\/190602-\/879f\_epoch\_100}
\DoxyCodeLine{}
\DoxyCodeLine{\$\ ./detectnet-\/console\ dog\_0.jpg\ output\_0.jpg\ \(\backslash\)}
\DoxyCodeLine{-\/-\/prototxt=\$NET/deploy.prototxt\ \(\backslash\)}
\DoxyCodeLine{-\/-\/model=\$NET/snapshot\_iter\_38600.caffemodel\ \(\backslash\)}
\DoxyCodeLine{-\/-\/input\_blob=data\ \(\backslash\)\ }
\DoxyCodeLine{-\/-\/output\_cvg=coverage\ \(\backslash\)}
\DoxyCodeLine{-\/-\/output\_bbox=bboxes}

\end{DoxyCode}


\begin{quote}
{\bfseries{note:\+}} the {\ttfamily input\+\_\+blob}, {\ttfamily output\+\_\+cvg}, and {\ttfamily output\+\_\+bbox} arguments may be omitted if your Detect\+Net layer names match the defaults above (i.\+e. if you are using the prototxt from following this tutorial). These optional command line parameters are provided if you are using a customized Detect\+Net with different layer names. \end{quote}


\hypertarget{md_jetson-inference_2docs_2detectnet-console_autotoc_md191}{}\doxysubsubsection{\texorpdfstring{Launching With a Pretrained Model}{Launching With a Pretrained Model}}\label{md_jetson-inference_2docs_2detectnet-console_autotoc_md191}

\hypertarget{md_jetson-inference_2docs_2detectnet-console_autotoc_md195}{}\doxysubsubsection{\texorpdfstring{Running Different Detection Models}{Running Different Detection Models}}\label{md_jetson-inference_2docs_2detectnet-console_autotoc_md195}
You can specify which model to load by setting the {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag on the command line to one of the corresponding CLI arguments from the table above. By default, Ped\+Net is loaded (pedestrian detection) if the optional {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag isn\textquotesingle{}t specified.
