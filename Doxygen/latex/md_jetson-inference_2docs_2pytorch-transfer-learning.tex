 \textsuperscript{Transfer Learning}\hypertarget{md_jetson-inference_2docs_2pytorch-transfer-learning_autotoc_md385}{}\doxysection{\texorpdfstring{Transfer Learning with Py\+Torch}{Transfer Learning with Py\+Torch}}\label{md_jetson-inference_2docs_2pytorch-transfer-learning_autotoc_md385}
Transfer learning is a technique for re-\/training a DNN model on a new dataset, which takes less time than training a network from scratch. With transfer learning, the weights of a pre-\/trained model are fine-\/tuned to classify a customized dataset. In these examples, we\textquotesingle{}ll be using the \href{https://arxiv.org/abs/1512.03385}{\texttt{Res\+Net-\/18}} and \doxysectlink{md_jetson-inference_2docs_2pytorch-ssd}{SSD-\/\+Mobilenet}{0} networks, although you can experiment with other networks too.

\href{https://arxiv.org/abs/1512.03385}{\texttt{}}

Although training is typically performed on a PC, server, or cloud instance with discrete GPU(s) due to the often large datasets used and the associated computational demands, by using transfer learning we\textquotesingle{}re able to re-\/train various networks onboard Jetson to get started with training and deploying our own DNN models. ~\newline


\href{https://pytorch.org/}{\texttt{Py\+Torch}} is the machine learning framework that we\textquotesingle{}ll be using, and example datasets along with training scripts are provided to use below, in addition to a camera-\/based tool for collecting and labeling your own training datasets. ~\newline

Note that the torch version should be reported as {\ttfamily 1.\+6.\+0} and the torchvision version should be {\ttfamily 0.\+7.\+0}.\hypertarget{md_jetson-inference_2docs_2pytorch-transfer-learning_autotoc_md388}{}\doxysubsection{\texorpdfstring{Mounting Swap}{Mounting Swap}}\label{md_jetson-inference_2docs_2pytorch-transfer-learning_autotoc_md388}
Unless you are on Jetson AGX Xavier, you should mount 4GB of swap space, as training uses up a lot of extra memory. ~\newline


Run these commands on your Jetson (outside of container) to disable ZRAM and create a swap file:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{sudo\ systemctl\ disable\ nvzramconfig}
\DoxyCodeLine{sudo\ fallocate\ -\/l\ 4G\ /mnt/4GB.swap}
\DoxyCodeLine{sudo\ mkswap\ /mnt/4GB.swap}
\DoxyCodeLine{sudo\ swapon\ /mnt/4GB.swap}

\end{DoxyCode}


Then add the following line to the end of {\ttfamily /\+etc/\+fstab} to make the change persistent:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{/mnt/4GB.swap\ \ none\ \ swap\ \ sw\ 0\ \ 0}

\end{DoxyCode}


Now your swap file will automatically be mounted after reboots. To check the usage, run {\ttfamily swapon -\/s} or {\ttfamily tegrastats}. Disabling ZRAM (in-\/memory compressed swap) also free\textquotesingle{}s up physical memory and requires a reboot to take effect.\hypertarget{md_jetson-inference_2docs_2pytorch-transfer-learning_autotoc_md389}{}\doxysubsection{\texorpdfstring{Disabling the Desktop GUI}{Disabling the Desktop GUI}}\label{md_jetson-inference_2docs_2pytorch-transfer-learning_autotoc_md389}
If you are running low on memory while training, you may want to try disabling the Ubuntu desktop GUI while you are training. This will free up extra memory that the window manager and desktop uses (around \texorpdfstring{$\sim$}{\string~}800\+MB for Unity/\+\+GNOME or \texorpdfstring{$\sim$}{\string~}250\+MB for LXDE) ~\newline


You can disable the desktop temporarily, run commands in the console, and then re-\/start the desktop when you are done training:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ sudo\ init\ 3\ \ \ \ \ \#\ stop\ the\ desktop}
\DoxyCodeLine{\#\ log\ your\ user\ back\ into\ the\ console}
\DoxyCodeLine{\#\ run\ the\ PyTorch\ training\ scripts}
\DoxyCodeLine{\$\ sudo\ init\ 5\ \ \ \ \ \#\ restart\ the\ desktop}

\end{DoxyCode}


If you wish to make this persistent across reboots, you can use the follow commands to change the boot-\/up behavior:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ sudo\ systemctl\ set-\/default\ multi-\/user.target\ \ \ \ \ \#\ disable\ desktop\ on\ boot}
\DoxyCodeLine{\$\ sudo\ systemctl\ set-\/default\ graphical.target\ \ \ \ \ \ \#\ enable\ desktop\ on\ boot}

\end{DoxyCode}
