a type for a packed binary type 

This type is a type designed to carry binary data that appears in various serialized formats, such as CBOR\textquotesingle{}s Major Type 2, Message\+Pack\textquotesingle{}s bin, and BSON\textquotesingle{}s generic binary subtype. This type is NOT a part of standard JSON and exists solely for compatibility with these binary types. As such, it is simply defined as an ordered sequence of zero or more byte values.

Additionally, as an implementation detail, the subtype of the binary data is carried around as a {\ttfamily std::\+uint8\+\_\+t}, which is compatible with both of the binary data formats that use binary subtyping, (though the specific numbering is incompatible with each other, and it is up to the user to translate between them).

\href{https://tools.ietf.org/html/rfc7049}{\texttt{CBOR\textquotesingle{}s RFC 7049}} describes this type as:\+ \begin{quote}
Major type 2:\+ a byte string. \doxylink{requirements_The}{The} string\textquotesingle{}s length in bytes is represented following the rules for positive integers (major type 0). \end{quote}


\href{https://github.com/msgpack/msgpack/blob/master/spec.md\#bin-format-family}{\texttt{Message\+Pack\textquotesingle{}s documentation on the bin type family}} describes this type as:\+ \begin{quote}
Bin format family stores an byte array in 2, 3, or 5 bytes of extra bytes in addition to the size of the byte array. \end{quote}


\href{http://bsonspec.org/spec.html}{\texttt{BSON\textquotesingle{}s specifications}} describe several binary types; however, this type is intended to represent the generic binary type which has the description:\+ \begin{quote}
Generic binary subtype -\/ This is the most commonly used binary subtype and should be the \textquotesingle{}default\textquotesingle{} for drivers and tools. \end{quote}


None of these impose any limitations on the internal representation other than the basic unit of storage be some type of array whose parts are decomposable into bytes.

\doxylink{requirements_The}{The} default representation of this binary format is a {\ttfamily std::\+vector\texorpdfstring{$<$}{<}std::\+uint8\+\_\+t\texorpdfstring{$>$}{>}}, which is a very common way to represent a byte array in modern C++.\hypertarget{classnlohmann_1_1basic__json_autotoc_md39}{}\doxysubsubsubsubsection{\texorpdfstring{Default type}{Default type}}\label{classnlohmann_1_1basic__json_autotoc_md39}
\doxylink{requirements_The}{The} default values for {\itshape Binary\+Type} is {\ttfamily std::\+vector\texorpdfstring{$<$}{<}std::\+uint8\+\_\+t\texorpdfstring{$>$}{>}}\hypertarget{classnlohmann_1_1basic__json_autotoc_md40}{}\doxysubsubsubsubsection{\texorpdfstring{Storage}{Storage}}\label{classnlohmann_1_1basic__json_autotoc_md40}
Binary Arrays are stored as pointers in a \doxylink{classnlohmann_1_1basic__json}{basic\+\_\+json} type. That is, for any access to array values, a pointer of the type {\ttfamily binary\+\_\+t\texorpdfstring{$\ast$}{*}} must be dereferenced.\hypertarget{classnlohmann_1_1basic__json_autotoc_md41}{}\doxysubsubsubsubsection{\texorpdfstring{Notes on subtypes}{Notes on subtypes}}\label{classnlohmann_1_1basic__json_autotoc_md41}

a type for a packed binary type 

This type is a type designed to carry binary data that appears in various serialized formats, such as CBOR\textquotesingle{}s Major Type 2, Message\+Pack\textquotesingle{}s bin, and BSON\textquotesingle{}s generic binary subtype. This type is NOT a part of standard JSON and exists solely for compatibility with these binary types. As such, it is simply defined as an ordered sequence of zero or more byte values.

Additionally, as an implementation detail, the subtype of the binary data is carried around as a {\ttfamily std::\+uint8\+\_\+t}, which is compatible with both of the binary data formats that use binary subtyping, (though the specific numbering is incompatible with each other, and it is up to the user to translate between them).

\href{https://tools.ietf.org/html/rfc7049}{\texttt{CBOR\textquotesingle{}s RFC 7049}} describes this type as:\+ \begin{quote}
Major type 2:\+ a byte string. \doxylink{requirements_The}{The} string\textquotesingle{}s length in bytes is represented following the rules for positive integers (major type 0). \end{quote}


\href{https://github.com/msgpack/msgpack/blob/master/spec.md\#bin-format-family}{\texttt{Message\+Pack\textquotesingle{}s documentation on the bin type family}} describes this type as:\+ \begin{quote}
Bin format family stores an byte array in 2, 3, or 5 bytes of extra bytes in addition to the size of the byte array. \end{quote}


\href{http://bsonspec.org/spec.html}{\texttt{BSON\textquotesingle{}s specifications}} describe several binary types; however, this type is intended to represent the generic binary type which has the description:\+ \begin{quote}
Generic binary subtype -\/ This is the most commonly used binary subtype and should be the \textquotesingle{}default\textquotesingle{} for drivers and tools. \end{quote}


None of these impose any limitations on the internal representation other than the basic unit of storage be some type of array whose parts are decomposable into bytes.

\doxylink{requirements_The}{The} default representation of this binary format is a {\ttfamily std::\+vector\texorpdfstring{$<$}{<}std::\+uint8\+\_\+t\texorpdfstring{$>$}{>}}, which is a very common way to represent a byte array in modern C++.\hypertarget{classnlohmann_1_1basic__json_autotoc_md516}{}\doxysubsubsubsubsection{\texorpdfstring{Default type}{Default type}}\label{classnlohmann_1_1basic__json_autotoc_md516}

With a parser callback function, the result of parsing a JSON text can be influenced.
