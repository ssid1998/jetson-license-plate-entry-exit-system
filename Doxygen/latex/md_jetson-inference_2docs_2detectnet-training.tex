 \textsuperscript{Object Detection}\hypertarget{md_jetson-inference_2docs_2detectnet-training_autotoc_md222}{}\doxysection{\texorpdfstring{Locating Objects with Detect\+Net}{Locating Objects with Detect\+Net}}\label{md_jetson-inference_2docs_2detectnet-training_autotoc_md222}
\doxylink{requirements_The}{The} previous image recognition examples output class probabilities representing the entire input image. \doxylink{requirements_The}{The} second deep learning capability we\textquotesingle{}re highlighting in this tutorial is detecting objects, and finding where in the video those objects are located (i.\+e. extracting their bounding boxes). This is performed using a \textquotesingle{}\doxylink{classdetect_net}{detect\+Net}\textquotesingle{} -\/ or object detection /\+ localization network.

\doxylink{requirements_The}{The} \href{../c/detectNet.h}{\texttt{{\ttfamily detect\+Net}}} object accepts as input the 2D image, and outputs a list of coordinates of the detected bounding boxes. To train the object detection model, first a pretrained Image\+Net recognition model (like Googlenet) is used with bounding coordinate labels included in the training dataset in addition to the source imagery.

\doxylink{requirements_The}{The} following pretrained Detect\+Net models are included with the tutorial:\+


\begin{DoxyEnumerate}
\item {\bfseries{ped-\/100}} (single-\/class pedestrian detector)
\item {\bfseries{multiped-\/500}} (multi-\/class pedestrian + baggage detector)
\item {\bfseries{facenet-\/120}} (single-\/class facial recognition detector)
\item {\bfseries{coco-\/airplane}} (MS COCO airplane class)
\item {\bfseries{coco-\/bottle}} (MS COCO bottle class)
\item {\bfseries{coco-\/chair}} (MS COCO chair class)
\item {\bfseries{coco-\/dog}} (MS COCO dog class)
\end{DoxyEnumerate}

As with the previous examples, provided are a console program and a camera streaming program for using \doxylink{classdetect_net}{detect\+Net}.\hypertarget{md_jetson-inference_2docs_2detectnet-training_autotoc_md223}{}\doxysubsubsection{\texorpdfstring{Detection Data Formatting in DIGITS}{Detection Data Formatting in DIGITS}}\label{md_jetson-inference_2docs_2detectnet-training_autotoc_md223}
Example object detection datasets with include \href{http://www.cvlibs.net/datasets/kitti/eval_object.php}{\texttt{KITTI}}, \href{http://mscoco.org/}{\texttt{MS-\/\+COCO}}, and others. To use the KITTI dataset follow this \href{https://github.com/NVIDIA/DIGITS/blob/digits-4.0/digits/extensions/data/objectDetection/README.md}{\texttt{DIGITS object detection tutorial with KITTI}}.

Regardless of dataset, DIGITS uses KITTI metadata format for ingesting the detection bounding labels. These consist of text files with frame numbers corresponding to image filenames, including contents such as:\+

\DoxyCodeLine{dog\ 0\ 0\ 0\ 528.63\ 315.22\ 569.09\ 354.18\ 0\ 0\ 0\ 0\ 0\ 0\ 0}
\DoxyCodeLine{sheep\ 0\ 0\ 0\ 235.28\ 300.59\ 270.52\ 346.55\ 0\ 0\ 0\ 0\ 0\ 0\ 0}

\end{DoxyCode}


\href{https://github.com/NVIDIA/DIGITS/blob/digits-4.0/digits/extensions/data/objectDetection/README.md}{\texttt{Read more}} about the folder structure and KITTI label format that DIGITS uses. ~\newline

\end{DoxyItemize}\hypertarget{md_jetson-inference_2docs_2detectnet-training_autotoc_md227}{}\doxysubsubsubsection{\texorpdfstring{Selecting Detect\+Net Batch Size}{Selecting Detect\+Net Batch Size}}\label{md_jetson-inference_2docs_2detectnet-training_autotoc_md227}
Detect\+Net\textquotesingle{}s network default batch size of 10 consumes up to 12GB GPU memory during training. However by using the {\ttfamily Batch Accumulation} field, you can also train Detect\+Net on a GPU with less than 12GB memory. See the table below depending on the amount of GPU memory available in your DIGITS server:\+


\begin{DoxyTable}{3}{}{}{1}
\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}GPU Memory  &\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}Batch Size  &\SetCell{c,bg=\tableheadbgcolor,font=\bfseries}Batch Accumulation  \\

8GB  &\SetCell{c}5  &\SetCell{c}2  \\
12GB or larger  &\SetCell{c}{\ttfamily \+[\doxylink{group__network}{Networking} defaults]\+} (10)  &\SetCell{c}Leave blank (1)  \\
\end{DoxyTable}


If you\textquotesingle{}re training on a card with 12GB of memory or more, leave the {\ttfamily Batch Size} as the default and leave the {\ttfamily Batch Accumulation} blank. In the network area select the {\ttfamily Custom Network} tab and then copy/\+paste the contents of \href{../data/networks/detectnet.prototxt}{\texttt{{\ttfamily detectnet.\+prototxt}}}



\doxylink{requirements_The}{The} Detect\+Net prototxt is located at \href{https://github.com/dusty-nv/jetson-inference/blob/master/data/networks/detectnet.prototxt}{\texttt{{\ttfamily data/\+networks/\+detectnet.\+prototxt}}} in the repo.\hypertarget{md_jetson-inference_2docs_2detectnet-training_autotoc_md229}{}\doxysubsubsubsection{\texorpdfstring{Training the Model with Pretrained Googlenet}{Training the Model with Pretrained Googlenet}}\label{md_jetson-inference_2docs_2detectnet-training_autotoc_md229}
Since Detect\+Net is derived from Googlenet, it is strongly recommended to use pre-\/trained weights from Googlenet as this will help speed up and stabilize training significantly. Download the Googlenet model from \href{http://dl.caffe.berkeleyvision.org/bvlc_googlenet.caffemodel}{\texttt{here}} or by running the following command from your DIGITS server:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{wget\ http://dl.caffe.berkeleyvision.org/bvlc\_googlenet.caffemodel}

\end{DoxyCode}


Then specify the path to your Googlenet under the {\ttfamily Pretrained Model} field.

Select a GPU to train on and set a name and group for the model:\+


\begin{DoxyItemize}
\item Group Name {\ttfamily MS-\/\+COCO}
\item Model Name {\ttfamily Detect\+Net-\/\+COCO-\/\+Dog}
\end{DoxyItemize}

Finally, click the {\ttfamily Create} button at the bottom to begin training.

Leave the training job to run for a while, say 50 epochs, until the m\+AP ({\ttfamily Mean Average Precision}) plot begins to increase. Note that due to the way m\+AP is calculated by the Detect\+Net loss function, the scale of m\+AP isn\textquotesingle{}t necessarily 0-\/100, and even an m\+AP between 5 and 10 may indicate the model is functional. With the size of the example COCO datasets we are using, it should take a couple hours training on a recent GPU before all 100 epochs are complete.
