Up next we have a realtime object detection camera demo available for C++ and Python:\+


\begin{DoxyItemize}
\item \href{../examples/detectnet-camera/detectnet-camera.cpp}{\texttt{{\ttfamily detectnet-\/camera.\+cpp}}} (C++)
\item \href{../python/examples/detectnet-camera.py}{\texttt{{\ttfamily detectnet-\/camera.\+py}}} (Python)
\end{DoxyItemize}

Similar to the previous \doxysectlink{md_jetson-inference_2docs_2detectnet-console}{{\ttfamily detectnet-\/console}}{0} example, these camera applications use detection networks, except that they process a live video feed from a camera. {\ttfamily detectnet-\/camera} accepts 4 optional command-\/line parameters:\+


\begin{DoxyItemize}
\item {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag setting the classification model (default is Ped\+Net)
\begin{DoxyItemize}
\item See \href{detectnet-console.md\#pre-trained-detection-models-available}{\texttt{Pre-\/trained Detection Models Available}} for the networks available to use.
\end{DoxyItemize}
\item {\ttfamily -\/-\/\doxylink{group__camera}{Camera Capture}} flag setting the camera device to use
\begin{DoxyItemize}
\item MIPI CSI cameras are used by specifying the sensor index ({\ttfamily 0} or {\ttfamily 1}, ect.)
\item V4\+L2 USB cameras are used by specifying their {\ttfamily /\+dev/\+video} node ({\ttfamily /\+dev/\+video0}, {\ttfamily /\+dev/\+video1}, ect.)
\item \doxylink{requirements_The}{The} default is to use MIPI CSI sensor 0 ({\ttfamily -\/-\/\doxylink{group__camera}{Camera Capture}=0})
\end{DoxyItemize}
\item {\ttfamily -\/-\/width} and {\ttfamily -\/-\/height} flags setting the camera resolution (default is {\ttfamily 1280x720})
\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} resolution should be set to a format that the camera supports.
\begin{DoxyItemize}
\item Query the available formats with the following commands:\+ ~\newline
 {\ttfamily  bash \$ sudo apt-\/get install v4l-\/utils \$ v4l2-\/ctl -\/-\/list-\/formats-\/ext }
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}

You can combine the usage of these flags as needed, and there are additional command line parameters available for loading custom models. Launch the application with the {\ttfamily -\/-\/help} flag to recieve more info, or see the \href{../README.md\#code-examples}{\texttt{{\ttfamily Examples}}} readme.
