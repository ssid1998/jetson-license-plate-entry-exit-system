\doxylink{requirements_The}{The} first model that we\textquotesingle{}ll be re-\/training is a simple model that recognizes two classes:\+ cat or dog.



Provided below is an 800MB dataset that includes 5000 training images, 1000 validation images, and 200 test images, each evenly split between the cat and dog classes. \doxylink{requirements_The}{The} set of training images is used for transfer learning, while the validation set is used to evaluate classification accuracy during training, and the test images are to be used by us after training completes. \doxylink{requirements_The}{The} network is never directly trained on the validation and test sets, only the training set.

\doxylink{requirements_The}{The} images from the dataset are made up of many different breeds of dogs and cats, including large felines like tigers and mountain lions since the amount of cat images available was a bit lower than dogs. Some of the images also picture humans, which the detector is essentially trained to ignore as background and focus on the cat vs. dog content.

To get started, first make sure that you have \href{pytorch-transfer-learning.md\#installing-pytorch}{\texttt{Py\+Torch installed}} on your Jetson, then download the dataset below and kick off the training script. After that, we\textquotesingle{}ll test the re-\/trained model in Tensor\+RT on some static images and a live camera feed.\hypertarget{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md349}{}\doxysubsection{\texorpdfstring{Downloading the Data}{Downloading the Data}}\label{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md349}

This way the dataset won\textquotesingle{}t be lost when you shutdown the container.

Run {\ttfamily python3 train.\+py -\/-\/help} for more information about each option that\textquotesingle{}s available for you to use, including other networks that you can try with the {\ttfamily -\/-\/arch} flag.\hypertarget{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md351}{}\doxysubsubsection{\texorpdfstring{Training Metrics}{Training Metrics}}\label{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md351}
\doxylink{requirements_The}{The} statistics output above during the training process correspond to the following info:\+


\begin{DoxyItemize}
\item Epoch:\+ an epoch is one complete training pass over the dataset
\begin{DoxyItemize}
\item {\ttfamily Epoch:\+ \+[N]\+} means you are currently on epoch 0, 1, 2, ect.
\item \doxylink{requirements_The}{The} default is to run for 35 epochs (you can change this with the {\ttfamily -\/-\/epochs=N} flag)
\end{DoxyItemize}
\item {\ttfamily \+[N/\+625]\+} is the current image batch from the epoch that you are on
\begin{DoxyItemize}
\item Training images are processed in mini-\/batches to improve performance
\item \doxylink{requirements_The}{The} default batch size is 8 images, which can be set with the {\ttfamily -\/-\/batch=N} flag
\item Multiply the numbers in brackets by the batch size (e.\+g. batch {\ttfamily \+[100/\+625]\+} -\/\texorpdfstring{$>$}{>} image {\ttfamily \+[800/\+5000]\+})
\end{DoxyItemize}
\item Time:\+ processing time of the current image batch (in seconds)
\item Data:\+ disk loading time of the current image batch (in seconds)
\item Loss:\+ the accumulated errors that the model made (expected vs. predicted)
\item {\ttfamily Acc@1}:\+ the Top-\/1 classification accuracy over the batch
\begin{DoxyItemize}
\item Top-\/1, meaning that the model predicted exactly the correct class
\end{DoxyItemize}
\item {\ttfamily Acc@5}:\+ the Top-\/5 classification accuracy over the batch
\begin{DoxyItemize}
\item Top-\/5, meaning that the correct class was one of the Top 5 outputs the model predicted
\item Since this Cat/\+\+Dog example only has 2 classes (Cat and Dog), Top-\/5 is always 100\%
\item Other datasets from the tutorial have more than 5 classes, where Top-\/5 is valid
\end{DoxyItemize}
\end{DoxyItemize}

You can keep an eye on these statistics during training to gauge how well the model is trained, and if you want to keep going or stop and test. As mentioned above, you can restart training again later if you desire.\hypertarget{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md352}{}\doxysubsubsection{\texorpdfstring{Model Accuracy}{Model Accuracy}}\label{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md352}
On this dataset of 5000 images, training Res\+Net-\/18 takes approximately \texorpdfstring{$\sim$}{\string~}7-\/8 minutes per epoch on Jetson Nano, or around 4 hours to train the model to 35 epochs and 80\% classification accuracy. Below is a graph for analyzing the training progression of epochs versus model accuracy:\+



At around epoch 30, the Res\+Net-\/18 model reaches 80\% accuracy, and at epoch 65 it converges on 82.\+5\% accuracy. With additional training time, you could further improve the accuracy by increasing the size of the dataset (see the Generating More Data section below) or by trying more complex models.

By default the training script is set to run for 35 epochs, but if you don\textquotesingle{}t wish to wait that long to test out your model, you can exit training early and proceed to the next step (optionally re-\/starting the training again later from where you left off). You can also download this completed model that was trained for a full 100 epochs from here:\+

This {\ttfamily classification/\+models} directory is automatically \href{aux-docker.md\#mounted-data-volumes}{\texttt{mounted into the container}}, so your trained models will persist after the container is shutdown.\hypertarget{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md353}{}\doxysubsection{\texorpdfstring{Converting the Model to ONNX}{Converting the Model to ONNX}}\label{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md353}
To run our re-\/trained Res\+Net-\/18 model with Tensor\+RT for testing and realtime inference, first we need to convert the Py\+Torch model into \href{https://onnx.ai/}{\texttt{ONNX format}} so that Tensor\+RT can load it. ONNX is an open model format that supports many of the popular ML frameworks, including Py\+Torch, Tensor\+Flow, Tensor\+RT, and others, so it simplifies transferring models between tools.

Py\+Torch comes with built-\/in support for exporting Py\+Torch models to ONNX, so run the following command to convert our Cat/\+\+Dog model with the provided {\ttfamily onnx\+\_\+export.\+py} script:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{python3\ onnx\_export.py\ -\/-\/model-\/dir=models/cat\_dog}

\end{DoxyCode}


This will create a model called {\ttfamily resnet18.\+onnx} under {\ttfamily jetson-\/inference/\+python/\+training/\+classification/\+models/\+cat\+\_\+dog/\+}\hypertarget{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md354}{}\doxysubsection{\texorpdfstring{Processing Images with Tensor\+RT}{Processing Images with Tensor\+RT}}\label{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md354}

\hypertarget{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md357}{}\doxysubsection{\texorpdfstring{Generating More Data (Optional)}{Generating More Data (Optional)}}\label{md_jetson-inference_2docs_2pytorch-cat-dog_autotoc_md357}
\doxylink{requirements_The}{The} images from the Cat/\+\+Dog dataset were randomly pulled from a larger 22.\+5\+GB \href{https://drive.google.com/open?id=1LsxHT9HX5gM2wMVqPUfILgrqVlGtqX1o}{\texttt{subset of ILSCRV12}} by using the \href{../tools/cat-dog-dataset.sh}{\texttt{{\ttfamily cat-\/dog-\/dataset.\+sh}}} script. This first Cat/\+\+Dog dataset is intentionally kept smaller to keep the training time down, but by using this script you can re-\/generate it with additional images to create a more robust model. ~\newline


Larger datasets take more time to train, so you can proceed to the \doxysectlink{md_jetson-inference_2docs_2pytorch-plants}{next example}{0} awhile, but if you were to want to expand the Cat/\+\+Dog dataset, first download the source data from here:\+


\begin{DoxyItemize}
\item \href{https://drive.google.com/open?id=1LsxHT9HX5gM2wMVqPUfILgrqVlGtqX1o}{\texttt{https:\+/\+/\+drive.\+google.\+com/\+open?id=1\+Lsx\+HT9\+HX5g\+M2w\+MVq\+PUf\+ILgrq\+Vl\+Gtq\+X1o}}
\end{DoxyItemize}

After extracting this archive, edit \href{../tools/cat-dog-dataset.sh}{\texttt{{\ttfamily tools/\+cat-\/dog-\/dataset.\+sh}}} with the following modifications:\+


\begin{DoxyItemize}
\item Substitue the location of the extracted {\ttfamily ilsvrc12\+\_\+subset} in the {\ttfamily IMAGENET\+\_\+\+DIR} variable
\item Then create an empty folder somewhere for cat\+\_\+dog, and substitue that location in {\ttfamily OUTPUT\+\_\+\+DIR}
\item Change the size of the dataset by modifying {\ttfamily NUM\+\_\+\+TRAIN}, {\ttfamily NUM\+\_\+\+VAL}, and {\ttfamily NUM\+\_\+\+TEST} variables
\end{DoxyItemize}

\doxylink{requirements_The}{The} script creates subdirectories for train, val, and test underneath the {\ttfamily OUTPUT\+\_\+\+DIR}, and will then fill those directories with the specified number of images for each. Then you can train the model the same way as above, optionally using the {\ttfamily -\/-\/resume} and {\ttfamily -\/-\/epoch-\/start} flags to pick up training where you left off (if you don\textquotesingle{}t want to restart training from the beginning). Remember to re-\/export the model to ONNX after re-\/training.
