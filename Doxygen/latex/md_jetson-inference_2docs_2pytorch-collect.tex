 \textsuperscript{Transfer Learning -\/ Classification}\hypertarget{md_jetson-inference_2docs_2pytorch-collect_autotoc_md363}{}\doxysection{\texorpdfstring{Collecting your own Classification Datasets}{Collecting your own Classification Datasets}}\label{md_jetson-inference_2docs_2pytorch-collect_autotoc_md363}
In order to collect your own datasets for training customized models to classify objects or scenes of your choosing, we\textquotesingle{}ve created an easy-\/to-\/use tool called {\ttfamily camera-\/capture} for capturing and labeling images on your Jetson from live video:\+



\doxylink{requirements_The}{The} tool will create datasets with the following directory structure on disk:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{‣\ train/}
\DoxyCodeLine{\ \ \ \ •\ class-\/A/}
\DoxyCodeLine{\ \ \ \ •\ class-\/B/}
\DoxyCodeLine{\ \ \ \ •\ ...}
\DoxyCodeLine{‣\ val/}
\DoxyCodeLine{\ \ \ \ •\ class-\/A/}
\DoxyCodeLine{\ \ \ \ •\ class-\/B/}
\DoxyCodeLine{\ \ \ \ •\ ...}
\DoxyCodeLine{‣\ test/}
\DoxyCodeLine{\ \ \ \ •\ class-\/A/}
\DoxyCodeLine{\ \ \ \ •\ class-\/B/}
\DoxyCodeLine{\ \ \ \ •\ ...}

\end{DoxyCode}


where {\ttfamily class-\/A}, {\ttfamily class-\/B}, ect. will be subdirectories containing the data for each object class that you\textquotesingle{}ve defined in a class label file. \doxylink{requirements_The}{The} names of these class subdirectories will match the class label names that we\textquotesingle{}ll create below. These subdirectories will automatically be populated by the tool for the {\ttfamily train}, {\ttfamily val}, and {\ttfamily test} sets from the classes listed in the label file, and a sequence of JPEG images will be saved under each.

Note that above is the organization structure expected by the Py\+Torch training script that we\textquotesingle{}ve been using. If you inspect the Cat/\+\+Dog and Plant\+CLEF datasets, they\textquotesingle{}re also organized in the same way.\hypertarget{md_jetson-inference_2docs_2pytorch-collect_autotoc_md364}{}\doxysubsection{\texorpdfstring{Creating the Label File}{Creating the Label File}}\label{md_jetson-inference_2docs_2pytorch-collect_autotoc_md364}
Under {\ttfamily jetson-\/inference/\+python/\+training/\+classification/\+data}, create an empty directory for storing your dataset and a text file that will define the class labels (usually called {\ttfamily labels.\+txt}). \doxylink{requirements_The}{The} label file contains one class label per line, and is alphabetized (this is important so the ordering of the classes in the label file matches the ordering of the corresponding subdirectories on disk). As mentioned above, the {\ttfamily camera-\/capture} tool will automatically populate the necessary subdirectories for each class from this label file.

\DoxyCodeLine{triceratops}
\DoxyCodeLine{velociraptor}

\end{DoxyCode}

\hypertarget{md_jetson-inference_2docs_2pytorch-collect_autotoc_md366}{}\doxysubsection{\texorpdfstring{Collecting Data}{Collecting Data}}\label{md_jetson-inference_2docs_2pytorch-collect_autotoc_md366}
Below is the {\ttfamily Data Capture Control} window, which allows you to pick the desired path to the dataset and load the class label file that you created above, and then presents options for selecting the current object class and train/\+val/\+test set that you are currently collecting data for:\+



First, open the dataset path and class labels. \doxylink{requirements_The}{The} tool will then create the dataset structure discussed above (unless these subdirectories already exist), and you will see your object labels populated inside the {\ttfamily Current Class} drop-\/down. Leave the {\ttfamily Dataset Type} as Classification.

Then position the camera at the object or scene you have currently selected in the drop-\/down, and click the {\ttfamily Capture} button (or press the spacebar) when you\textquotesingle{}re ready to take an image. \doxylink{requirements_The}{The} images will be saved under that class subdirectory in the train, val, or test set. \doxylink{requirements_The}{The} status bar displays how many images have been saved under that category.



It\textquotesingle{}s recommended to collect at least 100 training images per class before attempting training. A rule of thumb for the validation set is that it should be roughly 10-\/20\% the size of the training set, and the size of the test set is simply dictated by how many static images you want to test on. You can also just run the camera to test your model if you\textquotesingle{}d like.

It\textquotesingle{}s important that your data is collected from varying object orientations, camera viewpoints, lighting conditions, and ideally with different backgrounds to create a model that is robust to noise and changes in environment. If you find that you\textquotesingle{}re model isn\textquotesingle{}t performing as well as you\textquotesingle{}d like, try adding more training data and playing around with the conditions.\hypertarget{md_jetson-inference_2docs_2pytorch-collect_autotoc_md367}{}\doxysubsection{\texorpdfstring{Training your Model}{Training your Model}}\label{md_jetson-inference_2docs_2pytorch-collect_autotoc_md367}
When you\textquotesingle{}ve collected a bunch of data, then you can try training a model on it, just like we\textquotesingle{}ve done before. \doxylink{requirements_The}{The} training process is the same as the previous examples, and the same Py\+Torch scripts are used:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\$\ cd\ jetson-\/inference/python/training/classification}
\DoxyCodeLine{\$\ python3\ train.py\ -\/-\/model-\/dir=models/<YOUR-\/MODEL>\ data/<YOUR-\/DATASET>}

\end{DoxyCode}


\begin{quote}
{\bfseries{note:\+}} if you run out of memory or your process is "{}killed"{} during training, try \href{pytorch-transfer-learning.md\#mounting-swap}{\texttt{Mounting SWAP}} and \href{pytorch-transfer-learning.md\#disabling-the-desktop-gui}{\texttt{Disabling the Desktop GUI}}. ~\newline
 ~~~~~~~~~ to save memory, you can also reduce the {\ttfamily -\/-\/batch-\/size} (default 8) and {\ttfamily -\/-\/workers} (default 2) \end{quote}


Like before, after training you\textquotesingle{}ll need to convert your Py\+Torch model to ONNX:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\$\ python3\ onnx\_export.py\ -\/-\/model-\/dir=models/<YOUR-\/MODEL>}

\end{DoxyCode}


\doxylink{requirements_The}{The} converted model will be saved under {\ttfamily models/\+\texorpdfstring{$<$}{<}YOUR-\/\+MODEL\texorpdfstring{$>$}{>}/\+resnet18.onnx}, which you can then load with the {\ttfamily imagenet} programs like we did in the previous examples:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{NET=models/<YOUR-\/MODEL>}
\DoxyCodeLine{DATASET=data/<YOUR-\/DATASET>}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ C++\ (MIPI\ CSI)}
\DoxyCodeLine{imagenet\ -\/-\/model=\$NET/resnet18.onnx\ -\/-\/input\_blob=input\_0\ -\/-\/output\_blob=output\_0\ -\/-\/labels=\$DATASET/labels.txt\ csi://0}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ Python\ (MIPI\ CSI)}
\DoxyCodeLine{imagenet.py\ -\/-\/model=\$NET/resnet18.onnx\ -\/-\/input\_blob=input\_0\ -\/-\/output\_blob=output\_0\ -\/-\/labels=\$DATASET/labels.txt\ csi://0}

\end{DoxyCode}


If you need to, go back and collect more data and re-\/train your model again. You can restart the training from where you left off using the {\ttfamily -\/-\/resume} and {\ttfamily -\/-\/epoch-\/start} flags (run {\ttfamily python3 train.\+py -\/-\/help} for more info). Then remember to re-\/export the model.
