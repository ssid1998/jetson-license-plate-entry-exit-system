 \textsuperscript{Object Detection}\hypertarget{md_jetson-inference_2docs_2detectnet-console-2_autotoc_md182}{}\doxysection{\texorpdfstring{Locating Objects with Detect\+Net}{Locating Objects with Detect\+Net}}\label{md_jetson-inference_2docs_2detectnet-console-2_autotoc_md182}
\doxylink{requirements_The}{The} previous recognition examples output class probabilities representing the entire input image. Next we\textquotesingle{}re going to focus on {\bfseries{object detection}}, and finding where in the frame various objects are located by extracting their bounding boxes. Unlike image classification, object detection networks are capable of detecting many different objects per frame.



\doxylink{requirements_The}{The} \href{../c/detectNet.h}{\texttt{{\ttfamily detect\+Net}}} object accepts an image as input, and outputs a list of coordinates of the detected bounding boxes along with their classes and confidence values. \href{../c/detectNet.h}{\texttt{{\ttfamily detect\+Net}}} is available to use from \href{https://rawgit.com/dusty-nv/jetson-inference/master/docs/html/python/jetson.inference.html\#detectNet}{\texttt{Python}} and \href{../c/detectNet.h}{\texttt{C++}}. See below for various pre-\/trained detection models available for download. \doxylink{requirements_The}{The} default model used is a \href{../data/networks/ssd_coco_labels.txt}{\texttt{91-\/class}} SSD-\/\+Mobilenet-\/v2 model trained on the MS COCO dataset, which achieves realtime inferencing performance on Jetson with Tensor\+RT.

As examples of using the {\ttfamily \doxylink{classdetect_net}{detect\+Net}} class, we provide sample programs for C++ and Python:\+


\begin{DoxyItemize}
\item \href{../examples/detectnet/detectnet.cpp}{\texttt{{\ttfamily detectnet.\+cpp}}} (C++)
\item \href{../python/examples/detectnet.py}{\texttt{{\ttfamily detectnet.\+py}}} (Python)
\end{DoxyItemize}

These samples are able to detect objects in images, videos, and camera feeds. For more info about the various types of input/\+output streams supported, see the \doxysectlink{md_jetson-inference_2docs_2aux-streaming}{Camera Streaming and Multimedia}{0} page.\hypertarget{md_jetson-inference_2docs_2detectnet-console-2_autotoc_md183}{}\doxysubsubsection{\texorpdfstring{Detecting Objects from Images}{Detecting Objects from Images}}\label{md_jetson-inference_2docs_2detectnet-console-2_autotoc_md183}
First, let\textquotesingle{}s try using the {\ttfamily detectnet} program to locates objects in static images. In addition to the input/\+output paths, there are some additional command-\/line options:\+


\begin{DoxyItemize}
\item optional {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag which changes the \href{detectnet-console-2.md\#pre-trained-detection-models-available}{\texttt{detection model}} being used (the default is SSD-\/\+Mobilenet-\/v2).
\item optional {\ttfamily -\/-\/\doxylink{group__overlay}{Overlay}} flag which can be comma-\/separated combinations of {\ttfamily box}, {\ttfamily lines}, {\ttfamily labels}, {\ttfamily conf}, and {\ttfamily none}
\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} default is {\ttfamily -\/-\/\doxylink{group__overlay}{Overlay}=box,labels,conf} which displays boxes, labels, and confidence values
\item \doxylink{requirements_The}{The} {\ttfamily box} option draws filled bounding boxes, while {\ttfamily lines} draws just the unfilled outlines
\end{DoxyItemize}
\item optional {\ttfamily -\/-\/alpha} value which sets the alpha blending value used during overlay (the default is {\ttfamily 120}).
\item optional {\ttfamily -\/-\/threshold} value which sets the minimum threshold for detection (the default is {\ttfamily 0.\+5}). ~\newline

\end{DoxyItemize}

If you\textquotesingle{}re using the \doxysectlink{md_jetson-inference_2docs_2aux-docker}{Docker container}{0}, it\textquotesingle{}s recommended to save the output images to the {\ttfamily images/\+test} mounted directory. These images will then be easily viewable from your host device under {\ttfamily jetson-\/inference/\+data/\+images/\+test} (for more info, see \href{aux-docker.md\#mounted-data-volumes}{\texttt{Mounted Data Volumes}}).

You can specify which model to load by setting the {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag on the command line to one of the corresponding CLI arguments from the table above. By default, SSD-\/\+Mobilenet-\/v2 if the optional {\ttfamily -\/-\/\doxylink{group__network}{Networking}} flag isn\textquotesingle{}t specified.

For example, if you chose to download SSD-\/\+Inception-\/v2 with the \href{building-repo-2.md\#downloading-models}{\texttt{Model Downloader}} tool, you can use it like so:\+
