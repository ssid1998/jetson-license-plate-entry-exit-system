To test a custom segmentation network model snapshot on the Jetson, use the command line interface to \href{../examples/segnet-console/segnet-console.cpp}{\texttt{{\ttfamily segnet-\/console}}} ~\newline


First, for convienience, set the path to your extracted snapshot into a {\ttfamily \$NET} variable:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ NET=20170421-\/122956-\/f7c0\_epoch\_5.0}
\DoxyCodeLine{}
\DoxyCodeLine{\$\ ./segnet-\/console\ drone\_0428.png\ output\_0428.png\ \(\backslash\)}
\DoxyCodeLine{-\/-\/prototxt=\$NET/deploy.prototxt\ \(\backslash\)}
\DoxyCodeLine{-\/-\/model=\$NET/snapshot\_iter\_22610.caffemodel\ \(\backslash\)}
\DoxyCodeLine{-\/-\/labels=\$NET/fpv-\/labels.txt\ \(\backslash\)}
\DoxyCodeLine{-\/-\/colors=\$NET/fpv-\/deploy-\/colors.txt\ \(\backslash\)}
\DoxyCodeLine{-\/-\/input\_blob=data\ \(\backslash\)\ }
\DoxyCodeLine{-\/-\/output\_blob=score\_fr}

\end{DoxyCode}


This runs the specified segmentation model on a test image downloaded with the repo.



In addition to the aerial model from this tutorial, the repo also includes pre-\/trained models on other segmentation datasets, including {\bfseries{\href{https://www.cityscapes-dataset.com/}{\texttt{Cityscapes}}}}, {\bfseries{\href{http://synthia-dataset.net/}{\texttt{SYNTHIA}}}}, and {\bfseries{\href{http://host.robots.ox.ac.uk/pascal/VOC/}{\texttt{Pascal-\/\+VOC}}}}.\hypertarget{md_jetson-inference_2docs_2segnet-console_autotoc_md414}{}\doxysubsection{\texorpdfstring{autotoc\+\_\+md414}{autotoc\+\_\+md414}}\label{md_jetson-inference_2docs_2segnet-console_autotoc_md414}
