Allocate ZeroCopy mapped memory, shared between CUDA and CPU.

This is a templated version for allocating images from vector types like uchar3, uchar4, float3, float4, ect. The overall size of the allocation will be calculated as dims.x * dims.y * sizeof(T).

Parameters
[out]ptrReturned pointer to the shared CPU/GPU memory.
[in]dimsint2 vector where width=dims.x and height=dims.y
[in]clearIf true (default), the memory contents will be filled with zeros.
Returns
true if the allocation succeeded, false otherwise.

◆ cudaMallocMapped()

cudaError_t cudaMallocMapped ( void ** ptr,
size_t size,
bool clear = true )
inline

Allocate mapped memory that shares the same physical memory between CPU and GPU, as is the case on Jetson devices with unified memory. This eliminates the need for memory copies (Zero Copy), although synchronization is still required so that both processors are not accessing the same memory simultaneously.

Parameters
[out]ptrReturned pointer to the shared memory, can be accessed from both the CPU and in CUDA kernels This memory should be released with cudaFreeHost()
[in]sizeSize (in bytes) of the shared memory to allocate.
[in]clearIf true (the default), the memory contents will be filled with zeros.
Note
This function is the same as cudaAllocMapped(), but returns cudaError_t instead of bool.
Returns
cudaSuccess on success, cudaError_t on failure.
