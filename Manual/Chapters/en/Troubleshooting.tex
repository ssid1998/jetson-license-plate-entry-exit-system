\chapter{Troubleshooting}

\section{Troubleshooting}

This section lists the most common issues that may occur while operating the \textbf{License Plate Detection System}, along with their likely causes and solutions.  
These checks help maintain main unit  stability, reduce downtime, and ensure uninterrupted license plate recognition.

\vspace{0.5cm}


% ==========================================================
\subsection{Hardware and Power Issues}

\begin{itemize}
	\item \textbf{main unit  does not power on:}
	\begin{itemize}
		\item Confirm the \textbf{5V / 4A USB-C adapter} is properly connected and the LED on the main unit is lit.  
		\item Test a different cable or wall socket.  
		\item Avoid using unpowered USB hubs to supply power.
	\end{itemize}
	
	\item \textbf{Device overheats or shuts down unexpectedly:}
	\begin{itemize}
		\item Ensure proper airflow and use a small fan if necessary.  
		\item Run \texttt{sudo tegrastats} to monitor temperature.  
		\item Keep ambient temperature below \textbf{40\,\textdegree C}.
	\end{itemize}
	
	\item \textbf{Camera not detected:}
	\begin{itemize}
		\item Run \texttt{ls /dev/video*} to verify detection.  
		\item Re-insert the camera or test it on another PC.  
		\item Use a main USB port, not a low-power one.
	\end{itemize}
\end{itemize}

	\item \textbf{Low frame rate (FPS):}
	\begin{itemize}
		\item Use a lighter model (YOLOv8-Nano) or reduce camera resolution to 640×480.  
		\item Run \texttt{sudo nvpmodel -m 0} and \texttt{sudo jetson\_clocks} for max performance.  
		\item Close any other GPU-intensive programs.
	\end{itemize}
	
	\item \textbf{OCR returns wrong text:}
	\begin{itemize}
		\item Clean the camera lens and ensure good lighting.  
		\item Adjust image brightness or threshold parameters.  
		\item Retrain the OCR model with regional plate samples if needed.
	\end{itemize}
\end{itemize}

\subsection{Database and Storage Issues}

\begin{itemize}
	\item \textbf{Database locked:}
	\begin{itemize}
		\item Close any parallel process using \texttt{plates.db}.  
		\item Run \texttt{sqlite3 plates.db "PRAGMA journal\_mode=WAL;"} for safer writes.  
		\item Restart the app to release file locks.
	\end{itemize}
\begin{figure}[h!] % h! = "here, force placement"
	\centering
	\includegraphics[width=1.1\textwidth]{General/DBcorrupt.jpeg}
	\label{fig:jetson_setup}
\end{figure}
	\item \textbf{Database missing or corrupted:}
	\begin{itemize}
		\item Restore from backup using \texttt{sqlite3 plates.db < backup.sql}.  
		\item Check free space with \texttt{df -h}.  
		\item Avoid unplugging power during database writes.
	\end{itemize}
	
	\item \textbf{Storage full:}
	\begin{itemize}
		\item Delete old logs from \texttt{/logs/}.  
		\item Run \texttt{sudo apt clean} to clear unused packages.  
		\item Expand SD card storage if necessary.
	\end{itemize}
\end{itemize}

% ==========================================================
\subsection{Network or Communication Issues}

\begin{itemize}
	\item \textbf{SSH connection fails:}
	\begin{itemize}
		\item Check IP with \texttt{ifconfig} or \texttt{hostname -I}.  
		\item Enable SSH using \texttt{sudo system ctl enable ssh}.  
		\item Inspect router or firewall restrictions.
	\end{itemize}
	
	\item \textbf{Data not syncing across multiple units:}
	\begin{itemize}
		\item Confirm all devices have the same network time (\texttt{ntpdate}).  
		\item Test connection via \texttt{ping <IP>}.  
		\item Use a central database or MQTT broker for multi-unit setups.
	\end{itemize}
\end{itemize}
