\chapter{Main Function}
\section{Main Function}

The \textbf{main function} of this system is the operational core that integrates the Jetson Nano’s computing capability with machine learning-based detection and data storage.  
It is responsible for managing the flow from hardware input to software output — converting raw visual data into recognized and logged license plate information automatically.  

At startup, the system initializes hardware interfaces, activates the USB camera, loads trained deep learning models into GPU memory, and begins continuous frame acquisition.  
Each captured frame is analyzed in real time using the YOLOv8 object detection model \cite{ultralytics2023yolo}, followed by optical character recognition through EasyOCR \cite{jaided2021easyocr}.  
The extracted license plate numbers are immediately written into an SQLite database for persistent logging.  
This design demonstrates how embedded systems such as the \textbf{NVIDIA Jetson Nano} can execute advanced AI workloads locally, without depending on cloud servers \cite{nvidia2024jetson}.  

The combination of GPU-accelerated inference, lightweight OCR processing, and local storage provides an efficient and scalable solution for smart parking, toll automation, and vehicle monitoring applications \cite{pathak2023edgeai}.

\subsection{System Architecture Overview}

The License Plate Detection System follows a structured operational pipeline as shown below.  
It begins with continuous video capture through the Microsoft LifeCam Show (USB 2.0), followed by object detection on the Jetson Nano and local database storage of results.  
The edge computing design minimizes network latency and enhances reliability during outdoor deployment.

Each block operates independently yet communicates through Python functions and buffer queues to ensure no frame loss during processing.

\subsection{JetPack OS Installation}

The system operates on \textbf{NVIDIA JetPack 5.1.2 (Ubuntu 20.04)}, which provides CUDA, cuDNN, and TensorRT libraries necessary for deep learning inference \cite{nvidia2024jetson}.  
JetPack acts as the primary OS and supports Python, OpenCV, and PyTorch environments optimized for edge computing.

\textbf{Installation Summary:}
\begin{enumerate}
	\item Install \textbf{NVIDIA SDK Manager} on your host system.
	\item Connect Jetson Nano via USB and select:
	\begin{itemize}
		\item Target: \textbf{Jetson Nano Developer Kit}
		\item Version: \textbf{JetPack 5.1.2 (L4T 35.4.1)}
	\end{itemize}
	\item Flash the microSD card and insert it into the Jetson Nano slot.
	\item Power on the device and complete the Ubuntu setup wizard.
	\item Verify installation using:
	\begin{verbatim}
		$ cat /etc/nv_tegra_release
		# R35 (release), REVISION: 4.1
	\end{verbatim}
\end{enumerate}

\subsection{Execution Workflow}

The main execution loop consists of five steps:
\begin{enumerate}
	\item Power on Jetson Nano and open a terminal window.
	\item Navigate to the working directory:
	\begin{verbatim}
		$ cd ~/Code/Micropython
	\end{verbatim}
	\item Launch the program:
	\begin{verbatim}
		$ python3 main.py
	\end{verbatim}
	\item The live video feed appears with bounding boxes on detected plates.
	\item Recognized text and confidence are recorded in \texttt{plates.db}.
\end{enumerate}

\subsection{Data Flow and Storage}

Each detection cycle creates a structured record stored in the local SQLite database.  
The database includes four attributes: Plate Number, Timestamp, Detection Confidence, and Image Path (optional snapshot).  
This lightweight design ensures portability across multiple Jetson devices.
