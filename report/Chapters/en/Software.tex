\section{Integrated Development Environment IDE}

An Integrated Development Environment (IDE) is a software tool that offers a comprehensive platform for software development. It encompasses various components such as a code editor, debugger, compiler, and additional utilities that aid in the development process. By incorporating features like code completion, syntax highlighting, and code analysis, an IDE enhances the efficiency of writing, testing, and debugging code. Additionally, it facilitates project file management, change tracking, and collaboration among team members. Numerous IDEs are available, each boasting its own unique features and capabilities.

There are many different IDEs available, each with its own set of features and capabilities. Some popular IDEs include:

\begin{enumerate}
	\item Pycharm
	\item Visual Studio Code
	\item Jupyter Notebook
	\item Spyder
	\item Eclipse
	\item IntelliJ IDEA
\end{enumerate}

PyCharm: PyCharm is a popular IDE for Python development. It includes features like code completion, debugging, and refactoring tools. \hfill \break

\section{Overview of the Operating System}

NVIDIA offers the \textit{JetPack} operating system, which is compatible with not only the Jetson Nano but also the broader Jetson family, encompassing the Jetson AGX Xavier and Jetson TX2. This operating system is built on a Linux kernel and incorporates the Debian Management Tool, allowing users to work with the system using standard Linux expertise. Notably, it provides robust support for a variety of machine learning modules.


\begin{itemize}
	\item CUDA, 
	\item TensorRT, 
	\item Computer Vision, 
	\item Deep Stream SDK.
\end{itemize}  

This enables development for different hardware systems. Thus, the performance can be scaled depending on the application. \cite{Artamonov:2018,jetpack:2021} The figure~\ref{JetsonSDK} shows the structure of the development environment with the modules mentioned and others. \section{Installing the operating system}

To install the operating system, it's essential to prepare a microSD card. An image file can be obtained from the NVIDIA website, which is then loaded onto a computer. A specialized program is used to write this image file to the microSD card, considering the JetPack file system, identical to that of Linux. Once this process is completed, the microSD card is ready for use with the Jetson Nano. Upon the initial system startup, configuration is required, including the choice of installing with or without a graphical user interface. The subsequent steps will provide a detailed explanation of this process.

\subsection{Writing the operating system to the microSD card}

Using a computer that has an internet connection, the operating system is loaded from the NVIDIA website \cite{JetsonUserManual:2020, JetPackOS:2021}. The operating system can be copied to a microSD card using the usual tools, e.g. Etcher, available from the website \textcolor{blue}{\url{https://www.balena.io/etcher/}}. \cite{JetsonGettingStarted:2019} 

First, the microSD card must be formatted:

\section{ Configuring with a Graphical User Interface and Initial Startup}

With the microSD card containing the operating system in hand, the Jetson Nano can be prepared for configuring and using the OS. There are two fundamental options to choose from:


\begin{itemize}
	\item Utilizing a graphical interface
	\item Operating without a graphical user interface
\end{itemize}

This choice must be made prior to the first startup, as it cannot be altered later without reinstalling the operating system. Such a change would necessitate redoing all settings and adding any software not included with the operating system.

\bigskip

This section provides instructions for configuring a graphical user interface.


\bigskip

Before powering on the Jetson Nano, some initial steps are necessary. Start by connecting a monitor via an HDMI cable, along with a keyboard and a mouse. Ensure that the Jetson Nano and an appropriate power supply unit are on standby. However, refrain from plugging in the power supply unit just yet, as it would cause the Jetson Nano to start automatically. Under these conditions, insert the prepared microSD card into the Jetson Nano. Now, the system can be powered on. Begin by switching on the monitor, then connect the power supply unit to the Jetson Nano. The green PWR LED will illuminate, and the system will commence automatically. The boot process may take a few minutes, after which system configuration must be performed during the initial startup.



\subsection{First start}

During the initial boot process, several settings need to be configured:
\begin{itemize}
	\item Firstly, the End User License Agreement (EULA) for NVIDIA's Jetson Nano software is presented, and this agreement should be accepted.
	\item Next, language preferences must be set, and the appropriate keyboard layout should be chosen to match the connected keyboard.
	\item A suitable time zone must also be selected.
	\item Subsequent settings will be requested at the next login, and the following recommendations are provided:
	\begin{itemize}
		\item Name: MSR-Lab
		\item Computer Name: JetsonNanoMSRLab
		\item User Name: msrjetson
		\item Password: MSR
		\item Login Type: Automatic Login
	\end{itemize}
	\item Lastly, the partition size should be specified, with the maximum size typically chosen for standard applications.
\end{itemize}


Finally, the size of the partition must be specified. For the standard application, the maximum size must be specified here.

\section{Setup without graphic interface}

When the Jetson Nano is employed as an edge computer, the typical options for connecting a monitor or keyboard are often not available. In such cases, the system needs to be configured to operate without a graphical interface, and this process is described in this section.

For this, the following components are required:
\begin{itemize}
	\item A microSD card with the operating system,
	\item Jetson Nano,
	\item 5V power supply with a jack plug,
	\item Ethernet cable,
	\item USB cable with a microUSB plug,
	\item PC,
	\item Ethernet connector.
\end{itemize}

As there is no interface installed on the Jetson Nano for direct visual interaction, it needs to be connected to a computer. Since the Jetson Nano also requires an internet connection, only the micro USB port is available for this purpose. Consequently, a power supply unit with a jack plug is needed in this scenario. Before plugging in the power supply, connect the Jetson Nano to the computer via a USB interface and establish an internet connection through an Ethernet cable. It's important not to connect a monitor. During the initial startup of the Jetson Nano, it will verify this configuration; if a monitor is connected, the graphical user interface will be installed. At this point, you can insert the microSD card into the Jetson Nano, and then proceed to power it on.



\section{Remote access to the Jetson Nano}

\subsection{Access via Linux or MacOS}

After switching on the Jetson Nano, search for the USB device in \SHELL{/dev} on the Linux or MacOS host. For example, this is \SHELL{tty.usbmodemFD133}. 
This device is then connected to using the terminal emulator \glqq screen\grqq{}:

\medskip

\SHELL{screen /dev/tty.usbmodemFD133 115200}

\subsection{Access via Windows}

Under Windows, the interface of the Jetson Nano is found as a COM port. With the programme \glqq putty\grqq{} the connection via the serial interface can be started.

\subsection{Configuration of the Jetson Nano}

If the connection is successful, the installation wizard \SHELL{oem-config} is used to set the required data. As with the installation with a graphical interface, the End User License Agreement (EULA) from NVIDIA for the Jetson Nano software is displayed first. This agreement must be accepted. Then the language must be set. The keyboard layout can also be selected. This should match the connected keyboard. A suitable time zone must also be selected. Then the settings must be made, which will be requested at the next login. The following labels are recommended:

\begin{itemize}
	\item setting of the name: \textbf{MSR-Lab}
	\item Designation of the computer name:\textbf{JetsonNanoMSRLab}
	\item User name creation: \textbf{msrjetson}
	\item Setting the password: \textbf{MSR}
	\item Login type: \textbf{automatic login}
\end{itemize}


\bigskip

After that, the Jetson Nano is available. You can then connect to the nano via ssh or putty:

\medskip

\SHELL{ssh elmar@JetsonNanoMSRLab}

\section{Switching on and activating Secure Shell}

\ac{ssh} enables access to the Jetson Nano even if it is not equipped with a keyboard, mouse and monitor. The basic requirement is that the Jetson Nano is connected to the network via Ethernet or WLAN. 

\subsection{Installation of SSH}

An installation of \ac{ssh} is not necessary, as it is included in the distribution of the Jetson Nano's operating system. This only needs to be activated.

\subsection{Activation of SSH}

To activate \ac{ssh}, the \ac{ssh} server is started. This is done by the following command:

\medskip

\SHELL{sudo /etc/init.d/ssh start}

\medskip

Since this is necessary with every use of \ac{ssh}, this process is automated:

\medskip

\SHELL{sudo update-rc.d ssh defaults}

\medskip

After this entry, \ac{ssh} is permanently available, even after a reboot. Access via \ac{ssh} to the Jetson Nano is then always possible.

\bigskip

Another possibility is to place an empty file named \glqq ssh\grqq{} in the root directory. If the file exists, \ac{ssh} is automatically activated at the next boot.

\bigskip

You can activate \ac{ssh} access via the system settings. To do this, open the configuration dialogue in the start menu > Settings > Configuration, switch to the tab \glqq Interfaces\grqq{} and set the item \glqq SSH\grqq{} to \glqq Enabled\grqq{}. After confirmation, \ac{ssh} is also permanently available. The following command is used to open the dialogue:

\medskip

\SHELL{sudo raspi-config}

\medskip

The Jetson Nano must then be restarted:

\medskip

\SHELL{sudo reboot}

\subsection{Access via user name and IP address}

For \ac{ssh} access, the user name and password of the Jetson Nanoss must be known. The user name and associated password are specified during configuration during the first start-up.

\bigskip

In order to use \ac{ssh}, the IP address of the Jetson Nano must be known. With the terminal command

\medskip

\SHELL{ifconfig}

\medskip

the current network settings including the IP address are output.

\subsection{\ac{ssh}-Client under Windows}

Since 2017, Windows has had an \ac{ssh} implementation based on OpenSSH, which is integrated into the PowerShell command line. To use it, the PowerShell must be called up via the Start menu and the following command entered:

\medskip

\SHELL{ssh benutzername@<IPAdresseDesJetsonNanos>}

\medskip

When connecting for the first time, the \ac{ssh} keys of the Jetson Nano must be confirmed with \glqq yes\grqq{}. After entering the user password, remote access can be performed.

\section{Remote Desktop}

If the Jetson Nano is to be operated without a monitor, keyboard and mouse, then a remote desktop access server called \SHELL{xrdp} should be used. The installation is simple:

\medskip

\SHELL{\$ sudo apt-get install xrdp}

\medskip

After the installation is complete, the Jetson Nano must be restarted. Once the reboot is complete, the \SHELL{nmap} command on the PC can be used to check whether the installation of \SHELL{xrdp} was successful:

\medskip

\SHELL{\$ nmap jetson}

\SHELL{Starting Nmap 7.70 ( https://nmap.org ) at 2019-04-13 01:39 BST}

\SHELL{Nmap scan report for jetson (192.168.1.118)}

\SHELL{Host is up (0.0025s latency).}

\SHELL{Not shown: 996 closed ports}

\SHELL{PORT     STATE SERVICE}

\SHELL{22/tcp   open  ssh}

\SHELL{111/tcp  open  rpcbind}

\SHELL{3389/tcp open  ms-wbt-server}

\SHELL{Nmap done: 1 IP address (1 host up) scanned in 1.25 seconds}

\SHELL{\$}

\medskip

Although Remote Desktop Protocol (RDP) is a proprietary protocol, Microsoft Viewer provides it free of charge for most platforms. This must be installed. For the Windows system, the desktop must be added in the Microsoft Remote Desktop programme: \glqq Add Desktop\grqq

\medskip

Once the settings have been configured, the session can be set in full screen mode by setting a reasonable resolution for the resulting windowed desktop.

If the PC is connected to the board via RDP, the desktop will look slightly different. This is because a standard Ubuntu desktop is displayed, running Gnome, rather than the older Unity-style desktop that is the default for L4T. This means that a separate desktop is created for the RDP. This desktop remains until the user logs off, closing the RDP window does not close the desktop. This means that the next time the RDP window is opened, the desktop will still be available.

However, it should be noted that one cannot be logged on to the physical desktop and open an RDP desktop at the same time.

If the Jetson Nano is accessed via a wireless network, in the Gnome application, under \glqq Settings\grqq{}, select \glqq Power\grqq{} from the menu on the left and ensure that \glqq Turn off Wi-Fi to save power\grqq{} is turned off.

Now the monitor, keyboard and mouse can be unplugged, s are no longer needed.
