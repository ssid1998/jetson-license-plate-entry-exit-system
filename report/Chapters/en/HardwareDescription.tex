\section{Introduction to Nvidia Jetson Nano}
The NVIDIA® Jetson NanoTM Developer Kit serves as an accessible AI computer for makers, learners, and developers, making the capabilities of modern artificial intelligence available on a low-power, user-friendly platform. It offers quick initiation with its inherent compatibility with a diverse array of popular peripherals, extensions, and pre-built projects.

The \textbf{NVIDIA\textsuperscript{\textregistered} Jetson Nano} is bolstered by the extensive \textbf{NVIDIA\textsuperscript{\textregistered} JetPack\textsuperscript{\texttrademark} SDK}, delivering the essential performance and capabilities needed for the execution of contemporary AI workloads, which are outlined below. JetPack includes the following:

\begin{enumerate}
	\item Full desktop Linux with NVIDIA drivers
	\item AI and Computer Vision libraries and APIs
	\item Developer tools
	\item Documentation and sample code
\end{enumerate}

\section{The Jetson Nano and the Competition}

System-on-a-chip (SoC) systems provide a cost-effective entry point for the development of low-power, autonomous deep-learning applications, primarily tailored for inference, which involves applying pre-trained machine learning models to actual data. NVIDIA's Jetson Nano, the smallest AI module in the Jetson family, exemplifies such an SoC system, offering both high processing speeds and moderate power consumption. In the lower price range, numerous affordable systems have recently entered the market, designed for developing machine learning applications in embedded systems. These systems feature specialized computational kernels for efficient task execution while maintaining low power consumption. \cite{JetsonGettingStarted:2019}

Intel has introduced an affordable option for developers, like the Neural Compute Stick, which can be easily connected to existing hardware such as a Raspberry Pi via USB. For approximately 70 euros, you can access a system with 16 cores that relies on Intel's low-power processor chip specialized for image processing known as Movidius. This Intel product utilizes the Movidius Myriad X Vision Processing Unit and is compatible with operating systems like Windows® 10, Ubuntu, or MacOS. \cite{IntelStickDataSheet:2019}

Google is targeting the same audience with the Google Edge TPU Boards, which are available for 80 euros. These boards feature a Tensor Processing Unit (TPU) with hardware similar to what's used in Google Cloud and their development environment for Jupyter notebooks, Colab. Additionally, the Edge TPU Board can also function as a USB 3 dongle.
\cite{Vincent:2018,GoogleTensorFlowModel:2019,GoogleCoral:2019}

Similar technology is present in the most recent iteration of Apple's iPhone, featuring the A12 Bionic chip equipped with 8 cores and the capability to perform 5 trillion operations per second. Samsung's Exynos 980 includes an integrated Neural Processing Unit (NPU), and Huawei's Kirin 990 boasts the presence of two NPUs simultaneously. \cite{SamsungExynos980:2020,HuaweiKirin990:2020}

\section{Applications of the Jetson Nano}

The Jetson Nano offers a multitude of applications, ranging from simple tasks akin to those achievable with devices like the Raspberry Pi. Furthermore, it can serve as a compact yet robust computer for conventional purposes, including hosting web servers, internet browsing, document creation, or even as a budget-friendly gaming system. \cite{Kurniawan:2021}

However, the true strength of the Jetson Nano lies in its seamless integration of artificial intelligence, made exceptionally accessible by the NVIDIA TensorRT software development kit. This kit facilitates high-performance deep learning inference, allowing pre-trained networks to be effortlessly implemented and adapted as they encounter new data pertinent to the specific application.

NVidia's \cite{JetsonProjects:2020} website showcases a diverse range of projects contributed by the Jetson community. Beyond simple teleoperation, the Jetson Nano empowers the development of applications such as collision avoidance, object tracking, and even the creation of small autonomous vehicles. These projects extend beyond non-commercial use and include industrial applications. For instance, there's the Adaptive Traffic Controlling System, which employs visual traffic congestion analysis to optimize traffic light timings and alleviate congestion. Another project aids individuals with visual or reading impairments by converting recognized text, both printed and handwritten, into synthesized speech using only the Jetson Nano and the Pi camera. TensorFlow 2.0 and Google Colab are utilized for model training. Furthermore, the Jetson Nano, along with pre-trained models, can be harnessed for sign language, face, and object recognition.

\subsection{Processing Power}


The Jetson Nano is equipped with 472 GFLOPs of processing power, making it well-suited for handling contemporary Artificial Intelligence (AI) algorithms. It has the capability to run multiple neural networks simultaneously and process data from numerous high-resolution sensors, rendering it a valuable choice for Network Video Recorders (NVRs) and intelligent gateways.
Its components include an NVIDIA MaxwellTM architecture GPU with 128 NVIDIA CUDA® cores, running at 0.5 TFLOPs (FP16), and a Quad-core ARM® Cortex®-A57 MPCore processor. With 4 GB of 64-bit LPDDR4 RAM operating at 1600MHz (25.6 GB/s) and 16 GB of storage (16 GB eMMC 5.1 Flash), it offers ample multitasking capability.

\subsection{Connectivity}

With the help of an add on chip, WiFi (802.11b/g/n) connection is possible. The connection can also be achieved with the help of a Ethernet cable. Based on the specification of the cable used (10/100/1000 BASE-T) the speed may vary.

\section{NVIDIA Jetson Nano Developer Kit}

The NVIDIA Jetson Nano is a Single-Board Computer (SBC) explicitly designed for the application of machine learning algorithms. It's constructed to provide direct access to its components, inputs, and outputs. However, to safeguard the computer, it's advisable to use a suitable enclosure.



Similar to the Raspberry Pi, the Jetson Nano displays all its components and outputs openly. While the Jetson Nano kit's packaging can be used to assemble the Jetson Nano, for enhanced protection, it's recommended to place it in an additional case. An illustration \ref{Jetson:Aufloesungen} in the book demonstrates the Jetson Nano's orientation, which should be noted for clarity, especially concerning digital inputs and outputs.


\section{Construction of the Jetson Nano}


The NVIDIA Jetson Nano measures approximately $70 \times 45$mm when not enclosed in a case. The board is prominently marked by a substantial heat sink. Beneath this heat sink lies the formidable NVIDIA Tegra X1 microchip, housing four ARM application processor cores and 128 graphics processing units (GPUs). In one illustration, the top view of the NVIDIA Jetson Nano showcases its interfaces, while another provides a side view with the sockets.



\subsection{External power supply}

The Jetson Nano necessitates an external power source, specifically a DC power supply rated at 5V and 2A. This power supply can be connected to either the 5V DC interface or the Micro USB interface. When opting for the 5V DC interface, it's essential to use a jumper to connect the two pins of J48, situated between the camera and power supply connector, where the connector features a 2.1mm diameter with the center denoting the positive pole. Conversely, when utilizing the Micro USB interface, the jumper should be disconnected.


\subsection{HDMI and display port}

While not obligatory for later use, a monitor proves highly beneficial during setup and for specific applications. An external monitor can be linked to either the HDMI interface or the display port, offering diverse resolutions, including up to 4K.

It's important to highlight that HDMI-to-DVI adapters are not compatible. Consequently, it is imperative to employ a monitor equipped with native HDMI or display port support. The supported image resolutions are outlined in the provided table.

\subsection{$\mu$USB interface}

An alternative power source, a power supply rated at 5V and 2A with a micro-USB ($\mu$USB) output, can also be connected to the $\mu$USB interface. The choice of $\mu$USB is due to its availability in the form of affordable power supplies. The maximum power consumption falls within the range of 5W to 10W, emphasizing the importance of selecting a high-quality power supply unit. Utilizing an incompatible power supply can result in the NVIDIA Jetson Nano failing to initiate. One practical solution is to opt for a power supply with a shorter cable, as this minimizes voltage drops.



\subsection{USB 3.0 interfaces}

The Jetson Nano has 4 USB 3.0 interfaces. The current consumption on a USB interface is limited to 100mA. If the connected device has a higher consumption, it must be equipped with an additional power supply. As a rule, the USB keyboard and mouse can be connected.


\subsection{Ethernet interface}

The Jetson Nano offers an Ethernet interface with an RJ45 connection that enables direct internet connectivity or communication with other computers. For Gigabit Ethernet, a patch cable can be connected to the router. The Jetson Nano supports Ethernet connections using 10/100/1000BASE-T. Alternatively, a WiFi connection can be established by using an optional USB dongle for wireless LAN (WLAN).


\subsection{Status LED}

The status LED serves as an indicator for the power status of the Jetson Nano. It will flash when the power supply provides either too little or too much power.




\subsection{GPIOs}

The Jetson Nano features an interface with 40 connections referred to as the J41 interface, which are designated as General Purpose Input Output (GPIO) pins. These GPIO pins allow for the attachment of custom circuits and the expansion of the board's functionality, enabling the connection of components like LEDs or motors. Additionally, the Jetson Nano provides other intriguing interfaces, including SPI, I2C, and I2S, offering a range of possibilities for diverse applications.


\subsection{CSI connection for a camera}

The Jetson Nano is also furnished with a CSI connector for cameras. It adheres to a precise specification, offering 12 lanes with configurations of either 3x4 or 4x2, complying with MIPI CSI-2 D-PHY 1.1 standards at a rate of 1.5 Gb/s per pair. With compatible software, the Jetson Nano can be harnessed for image recognition applications.

\subsection{microSD card}


The Jetson Nano lacks an onboard hard disk for the operating system, necessitating an alternative storage method. To address this, the Jetson Nano utilizes a microSD card, which houses both the operating system and application programs. Therefore, the selection of an appropriate SD card is crucial, with an emphasis on speed and capacity. A recommended minimum is a 16GB UHS-1 card. The card slot is located on the underside of the Jetson Nano.

\section{Jetson Nano Interfaces}
\subsection{J41 Expansion Header Pinout - GPIO Pins}
The Jetson Nano is capable of interfacing with other systems, and for the purpose of communication or external device control, it can be advantageous to have the ability to manage the pins on the J41 header.
The pins allow different types of communication: 


\begin{itemize}
	\item Serial communication Universal Asynchronous Receiver Transmitter \ac{uart}
	\item Communication via Inter-Integrated Circuit \ac{i2c}
	\item Serial Peripheral Interface \ac{spi}
	\item  Inter-IC Sound \ac{i2s}
	\item Direct communication via \ac{gpio}
\end{itemize}    

Binary signals are exchanged between two systems through GPIO communication, making them essentially digital inputs and outputs that can be set as required. By default, all interface signal pins on the Jetson Nano are configured as GPIO, with the exception of pins 3, 5, 8, 9, 27, and 28. Pins 3 and 5, along with pins 27 and 28, are dedicated to the two I2C interfaces, providing the Serial Data (SDA) and Serial Clock (SCL) lines. Pins 8 and 10 are allocated for the receive line (Rx) and transmit line (Tx) of the UART.To use the GPIO, the GPIO library must be installed. \cite{JetsonUserManual:2020} 

To use the \ac{gpio}, the GPIO library must be installed.\cite{GitHubGPIO:2020}.

This section highlights some of the Jetson Nano Developer Kit interfaces.
\begin{itemize}
	\item Module
	\begin{itemize}
		\item [J501] Slot for a microSD card.
		\item The passive heatsink supports 10W module power usage at 25° C ambient temperature. If your user case requires additional cooling, you can configure the	module to control a system fan.
	\end{itemize}
	\item Carrier Board
	\begin{itemize}
		\item (DS3) Power LED; lights when the developer kit is powered on.
		\item (J2) SO-DIMM connector for Jetson Nano module.
		\item (J6] HDMI and DP connector stack.
		\item (J13) Camera connector; enables use of CSI cameras. Jetson Nano Developer Kit works with IMX219 camera modules, including Leopard Imaging LI-IMX219-MIPIFF-NANO camera module and Raspberry Pi Camera Module V2.
		\item (J15) 4-pin fan control header. Pulse Width Modulation (PWM) output and tachometer input are supported.
		\item (J18) M.2 Key E connector can be used for wireless networking cards; includes	interfaces for PCIe (x1), USB 2.0, UART, I2S, and I2C. To reach J18 you must detach the Jetson Nano module.
		\item (J25) Power jack for 5V~4A power supply. Accepts a 2.1×5.5×9.5 mm plug with positive polarity.
		\item (J28) Micro-USB 2.0 connector; can be used in either of two ways:
		If J48 pins are not connected, you can power the developer kit from a 5V~2A
		Micro-USB power supply.
		If J48 pins are connected, operates in Device Mode.
		\item (J32 and J33) are each a stack of two USB 3.0 Type A connectors. Each stack is limited
		to 1A total power delivery. All four are connected to the Jetson Nano module via a
		USB 3.0 hub built into the carrier board.
		\item (J38) The Power over Ethernet (POE) header exposes any DC voltage present on J43 Ethernet jack per IEEE 802.3af.
		\item (J40) 8-pin button header; brings out several system power, reset, and force recovery related signals.
	\end{itemize}
\end{itemize}
