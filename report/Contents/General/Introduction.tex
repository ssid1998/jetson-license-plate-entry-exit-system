\chapter{Introduction}
\label{ch:introduction}

\begin{center}
	\textbf{Project: Car License Plate Identification on Jetson Nano}\\
	Edge deployment of a real-time \ac{LPR} pipeline (detection + \ac{OCR}) with local logging and dwell-time based decisions for high-throughput access and parking scenarios.
\end{center}

\vspace{0.4cm}

Automated vehicle identification is a core building block for modern transportation systems, ranging from access control at security gates to time-limit enforcement in airport drop-off zones and paid parking facilities. In these settings, throughput is high and decisions must be made quickly: a vehicle should be identified within fractions of a second to avoid congestion, while the system must remain robust to lighting variation, motion blur, and partial occlusions. Traditional \ac{ANPR} deployments often rely on centralized servers and proprietary camera systems, increasing cost and limiting flexibility for smaller sites or pilot installations \cite{anpr-survey-2020}.

This project implements a compact, \textbf{edge-first} license plate detection and recognition system designed to run directly on the \textbf{NVIDIA Jetson Nano} \cite{JetsonNano2022}. The core idea is simple and operationally motivated: detect a license plate at \textbf{entry} and \textbf{exit}, compute dwell time from timestamps, and present a decision (e.g., free exit, payment required, or threshold violation) locally without requiring cloud services. The resulting prototype combines a lightweight object detector (YOLO family) for plate localization \cite{Ultralytics2023}, \textbf{EasyOCR} for text extraction \cite{easyocr-2020}, validation logic tailored to German plates, and a local \textbf{SQLite} database for persistent state.

\section{Problem Description}
There are multiple problems while managing operations in places where vehicular traffic is high, such as airports, parking-lots and logistics parks. Manual verification thus becomes prone to errors, expends time, and has coherency issues, along with commercial \ac{ANPR} cameras being expensive and having high maintenance.  \cite{netinant2024evaluating}. Airports, for example,  require systems that are fast, less error prone while being compliant under the purview of legality and warrant the need for vehicle identification systems operable in differing conditions such as light intensity, weather variations, and types of license plates \cite{fraportALPR, munichALPR}.

Key issues include:
\begin{itemize}
	\item \textbf{High Throughput:} Thousands of entries per day demand sub-second recognition to prevent congestion.
	\item \textbf{Diverse Plate Formats:} Germany’s 15+ plate variants (electric, diplomatic, export, etc.) pose recognition challenges \cite{mufti2021automatic}.
	\item \textbf{Environmental Variations:} Lighting changes, glare, and weather affect detection reliability \cite{plavac2024performance}.
\end{itemize}

These constraints necessitate a locally deployed, energy-efficient, and privacy-preserving \ac{AI} system capable of performing all detection and billing logic directly on edge hardware.

\section{Challenges}
Building a practical edge \ac{LPR} prototype required addressing constraints that do not show up in offline model evaluation:
\begin{itemize}
	\item \textbf{Compute and memory limits:} the Jetson Nano (4\,GB unified memory) must run video capture, detection, \ac{OCR}, and UI concurrently. Unbounded buffering or running \ac{OCR} on every frame quickly becomes infeasible.
	\item \textbf{Real-world imaging conditions:} glare, shadows, night lighting, motion blur, and oblique viewing angles reduce \ac{OCR} quality even if detection is correct.
	\item \textbf{Plate-like false positives:} vehicle grills, bumper stickers, and reflections can trigger spurious detections, requiring confidence thresholds and domain validation.
	\item \textbf{Noisy \ac{OCR} outputs:} common confusions (O/0, B/8, missing separators) require normalization and temporal smoothing before committing results to persistent storage.
	\item \textbf{Long-run robustness:} a gate system must run continuously; storage paths (external SSD), database integrity, and recovery after restarts influence usability as much as model choice.
\end{itemize}

\section{Solution Description}
To simplify the challenges mentioned, the project incorporates combined usage of hardware optimization, easy and efficient \ac{AI} design, and enables compliance within legal frameworks.  
The methods defined directly map to challenges identified, while promoting technical robustness and ensuring alignment within regulatory frameworks.

\paragraph{Addressing Environmental and Lighting Variability}
\begin{itemize}
	\item Apply dynamic image preprocessing using \textbf{OpenCV} to normalize brightness, gamma, and contrast across frames.
	\item Use deraining and dehazing filters to reduce weather-induced distortions before \textbf{YOLO} inference.
	\item Incorporate datasets with a great variety in daytime, nightime, and fluctuating weather conditions, aiding in improving accuracy.
\end{itemize}

\paragraph{Optimizing Computational Performance}
\begin{itemize}
	\item Quantize \textbf{YOLO} weights to \textbf{FP16} and optimize inference through TensorRT to achieve real-time speed.
	\item Use a hybrid configuration—\textbf{YOLO-Nano} for continuous inference and \textbf{YOLO-Large} for periodic re-validation of uncertain detections.
	\item Usage of multi-threaded and asynchronous system architecture pipelines to enable the parallelization of camera feed capture and \ac{OCR}.
	\item Monitor \ac{GPU} utilization via \textbf{Jetson-Stats} for automatic frame-rate adjustments under thermal constraints.
\end{itemize}

\paragraph{Reducing False Positives and Handling Non-Standard Plates}
\begin{itemize}
	\item Introduce temporal validation across consecutive frames to confirm detection stability.
	\item Apply bounding-box correlation between vehicle and plate regions to prevent spurious matches.
\end{itemize}

\paragraph{Improving Cost Efficiency and Scalability}
\begin{itemize}
	\item Use open-source tools (\textbf{YOLO}, \textbf{EasyOCR}, \textbf{SQLite}) to eliminate proprietary license costs.
	\item Develop a modular architecture separating hardware (camera), \ac{AI} inference, and database layers for flexible scaling.
\end{itemize}

\subsection{Solution System Overview}
The system follows a multi-stage process described as follows:
\begin{itemize}
	\item \textbf{Video capture:} OpenCV captures frames from a USB camera (Microsoft LifeCam used in our setup).
	\item \textbf{Plate detection:} a YOLO-based detector localizes license plates in each frame (Ultralytics YOLO training artefacts are included in the repository) \cite{Ultralytics2023}.
	\item \textbf{Plate crop and preprocessing:} the detected region is cropped, normalized, and optionally masked to handle the EU strip region.
	\item \textbf{\ac{OCR}:} EasyOCR reads the cropped plate image and returns candidate strings with confidence scores \cite{easyocr-2020}.
	\item \textbf{Temporal smoothing:} repeated reads across frames are combined into a stable consensus to reduce flicker and one-off errors.
	\item \textbf{German plate validation:} regex-based format checks and visual heuristics (e.g., EU blue strip detection) reject implausible outputs.
	\item \textbf{Local persistence and billing:} validated plates are written to an SQLite database; dwell time is derived from the first/last detection and converted into an operator-defined fee or decision.
	\item \textbf{Operator UI:} a local dashboard displays the annotated video stream and a summary of recent detections and calculated fees.
\end{itemize}
