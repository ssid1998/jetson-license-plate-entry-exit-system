\section{Hardware Verification}

\subsection{Introduction}
An important prerequisite to deploying Deep Learning Models such as YOLOv8 and Optical Character Recognition (OCR) is ensuring the physical layer is reliable. 
In the context of this project, the hardware architecture comprises an edge computing unit along with an optical acquisition unit. Before software pipeline deployment, these components must be verified systematically—especially the Seeed Studio reComputer J1010 and the Microsoft LifeCam interface. 

This chapter documents the utilized hardware components, defining their specific functions within the License Plate Recognition (LPR) architecture, while detailing the rigorous testing protocols applied for operational status verification.

\subsection{Parts and Specification}

The system builds upon two hardware pillars: the processing unit (Jetson Nano) and the image acquisition unit (Microsoft LifeCam).

\paragraph{Processing Unit: reComputer J1010}
The core processing unit used is the \textbf{Seeed Studio reComputer J1010}. The device is an embedded edge AI box consisting of the NVIDIA Jetson Nano module (4GB B01). It is selected primarily due to its ability to perform CUDA computations in parallel, which is an important prerequisite for real-time YOLO inference. This enables the system to follow a low-power profile.

\paragraph{Optical Unit: Microsoft LifeCam}
For image acquisition, the \textbf{Microsoft LifeCam} was selected. This is a high-definition USB webcam capable of capturing the detailed frames necessary for accurate character segmentation during the OCR phase.

\begin{itemize}
	\item \textbf{Sensor Resolution:} 2.0 Megapixel (1080p / 720p video modes).
	\item \textbf{Lens Type:} Wide-angle glass element lens with high-precision optics.
	\item \textbf{Interface:} USB 2.0 High Speed.
	\item \textbf{Focus:} Auto-focus capability (critical for focusing on moving vehicles).
	\item \textbf{Mounting:} Flexible attachment base for positioning on tripods or monitors.
\end{itemize}

\subsection{Hardware Functions}

Each component plays a distinct, interconnected role in the LPR pipeline. The data flow between these hardware parts is critical for system latency and accuracy.

\begin{enumerate}
	\item \textbf{Image Acquisition (Camera):} The Microsoft LifeCam captures raw video frames of vehicles. Primarily, the hardware function is to generate and buffer video frames, which are transmitted through the USB bus to the Jetson Nano's memory controller.
	\item \textbf{Preprocessing (CPU):} The ARM Cortex A57 CPU is responsible for initial resizing and normalization of the image frame (e.g., resizing $1920 \times 1080$ to $640 \times 640$) to match the input tensor size required by the YOLO network.
	\item \textbf{Inference (GPU):} The 128-core Maxwell GPU executes the forward pass of the YOLO convolutional neural network. This is computationally intensive, requiring CUDA cores to perform matrix multiplications in parallel to identify license plate bounding boxes.
\end{enumerate}

\subsection{Hardware Testing Documentation}

To ensure the hardware supports the software requirements described in previous chapters, specific test cases were devised. The objective was to validate the operational integrity of the GPU, Camera interface, Thermal Stability, and Automation Services.

\subsubsection{Test 1: GPU and Environment Verification}
Since the YOLOv8 model relies heavily on GPU acceleration to achieve real-time frame rates (FPS), verifying the GPU availability is the first step. A Python script leveraging the PyTorch library queries the hardware status to validate that the JetPack OS is correctly installed and CUDA libraries are accessible.

\textbf{Test Procedure:}
\begin{enumerate}
	\item Power and boot the reComputer J1010.
	\item Navigate to the testing directory via the terminal.
	\item Execute the Python Verification script.
\end{enumerate}

The following code, \texttt{test\_gpu.py}, was executed on the system:

\begin{lstlisting}[language=Python, caption={Python Script for CUDA and GPU Verification}, label={lst:gpu_test}]
	## @file test_gpu.py
	#  @brief Verifies CUDA and GPU availability for YOLOv8 on Jetson Nano.
	import torch
	import sys
	
	def verify_system():
	print("--- Hardware Verification: Jetson Nano ---")
	
	# 1. Check Python Version
	print(f"Python Version: {sys.version}")
	
	# 2. Check CUDA Availability for YOLO
	cuda_available = torch.cuda.is_available()
	print(f"CUDA Available: {cuda_available}")
	
	if cuda_available:
	# 3. Get Device Properties
	device = torch.device("cuda")
	props = torch.cuda.get_device_properties(device)
	print(f"Device Name: {props.name}")
	print(f"Total Memory: {props.total_memory / 1e9:.2f} GB")
	
	# 4. Perform a Tensor Calculation Test
	try:
	x = torch.rand(5, 3).to(device)
	print("Tensor Allocation Test: SUCCESS")
	print(f"Test Tensor: \n{x}")
	except Exception as e:
	print(f"Tensor Allocation Test: FAILED - {e}")
	else:
	print("CRITICAL WARNING: System is running on CPU only.")
	
	if __name__ == "__main__":
	verify_system()
\end{lstlisting}

\textbf{Results of Test:} \\
The output confirmed the correct configuration of the reComputer J1010. The system identified the "NVIDIA Tegra X1" device and performed a successful test tensor allocation. Memory checks reported roughly 3.9 GB of accessible memory, confirming readiness for inference.

The test code, adapted from \texttt{LifeCam.py}, initializes the camera at index 0 and streams video.

\begin{lstlisting}[language=Python, caption={OpenCV Script for Camera Hardware Testing}, label={lst:cam_test}]
	## @file test_camera.py
	#  @brief Tests the Microsoft LifeCam hardware interface.
	import cv2
	import time
	
	def test_camera_hardware(source=0):
	print(f"--- Testing Camera Source: {source} ---")
	
	# 1. Initialize VideoCapture
	cap = cv2.VideoCapture(source)
	cap.set(cv2.CAP_PROP_FRAME_WIDTH, 640)
	cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 480)
	
	if not cap.isOpened():
	print("ERROR: Could not open camera hardware.")
	return
	
	# 2. Warm-up camera
	print("Warming up sensor...")
	time.sleep(2)
	
	# 3. Capture Loop
	print("Starting video stream. Press 'q' to exit.")
	while True:
	ret, frame = cap.read()
	if ret:
	cv2.imshow("Hardware Test: LifeCam", frame)
	if cv2.waitKey(1) & 0xFF == ord('q'):
	break
	else:
	print("Frame Capture: FAILED")
	break
	
	# 4. Release Resource
	cap.release()
	cv2.destroyAllWindows()
	print("Camera resource released.")
	
	if __name__ == "__main__":
	test_camera_hardware(0)
\end{lstlisting}

\textbf{Test Results:} \\
The script successfully initialized the Microsoft LifeCam. The video feed confirmed that the USB bandwidth was sufficient to display a stream without frame loss. Image clarity was visually verified to ensure it met OCR requirements.

\subsubsection{Test 3: Thermal Stability and Power Consumption}

Deep learning inference induces significant thermal load on the Tegra X1 SoC. To ensure the passive heatsink provides sufficient cooling, a stress test was conducted.

\textbf{Test Procedure:}
\begin{enumerate}
	\item Activate maximum performance mode: \texttt{sudo nvpmodel -m 0}.
	\item Maximize clock speeds: \texttt{sudo jetson\_clocks}.
	\item Monitor core metrics using \texttt{tegrastats}.
	\item Run the YOLOv8 inference script for 10 minutes continuously.
\end{enumerate}

\textbf{Test Results:} \\
The system stabilized at an operating temperature of roughly $45^\circ$C, well below the $85^\circ$C throttling threshold. Power consumption peaked at \SI{8.2}{\watt}, confirming the \SI{20}{\watt} (5V/4A) power supply provides ample headroom.

\subsubsection{Test 4: Automated Startup and Service Recovery}

To validate the "Plug-and-Play" requirement for airport deployment, the system must launch automatically upon boot and recover from crashes without intervention.

\textbf{Test Procedure:}
\begin{enumerate}
	\item \textbf{Cold Boot Test:} Remove power, reconnect, and measure time to active detection.
	\item \textbf{Service Status Check:} Verify background service using \texttt{systemctl status lpr-system}.
	\item \textbf{Crash Simulation:} Manually terminate the process (\texttt{kill -9 PID}) to test the auto-restart policy.
\end{enumerate}

\begin{lstlisting}[language=bash, caption={Automation Verification Script}, label={lst:auto_test}]
	#!/bin/bash
	SERVICE_NAME="lpr-system.service"
	echo "--- Automation Test: Service Health Check ---"
	
	# 1. Check if Service is Active
	if systemctl is-active --quiet $SERVICE_NAME; then
	echo "[PASS] Service is currently RUNNING."
	else
	echo "[FAIL] Service is INACTIVE."
	exit 1
	fi
	
	# 2. Crash Simulation & Recovery Check
	PID=$(pgrep -f "main_detection.py")
	kill -9 $PID
	sleep 5
	NEW_PID=$(pgrep -f "main_detection.py")
	
	if [ "$NEW_PID" != "$PID" ] && [ ! -z "$NEW_PID" ]; then
	echo "[PASS] Service successfully RESTARTED. New PID: $NEW_PID"
	else
	echo "[FAIL] Service did not restart."
	fi
\end{lstlisting}

\textbf{Test Results:} \\
The system successfully initialized the detection pipeline 45 seconds after power-on. The \texttt{systemd} configuration successfully rebooted the application within 3 seconds of a forced crash.

The hardware verification phase confirmed that the reComputer J1010 and Microsoft LifeCam are fully interoperable. The GPU is accessible for deep learning, the thermal solution manages heat effectively under load, and the automation scripts ensure high availability.
