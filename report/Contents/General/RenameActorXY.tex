\chapter{Display as System Actor}
\label{chap:display_actor}

\section{Introduction}
In embedded AI systems, \textbf{actors} are components that convert computational results into observable system responses.  
In this project, the actor is implemented as a \textbf{visual display interface} connected to the NVIDIA Jetson Nano, presenting real-time recognition outcomes such as \textit{“Willkommen”} (Welcome) or \textit{“Ausfahrt registriert”} (Exit recorded) when a vehicle’s license plate is successfully detected.  
This visual feedback replaces traditional actuators like buzzers or gates, offering clear, non-intrusive communication to the user \cite{edgeAIbook, opencv_doc_2024}.  

The display used is attached to the Jetson Nano’s \textbf{HDMI 2.0 (J3)} port on the Seeed Studio J101 carrier board.  
It functions both as a debugging interface during development and as the final system actor during runtime.

\section{General}
Displays serve as standard output actors in access-control and surveillance systems, allowing real-time visual verification of detections without additional hardware.  
In this setup, the HDMI-connected monitor provides a direct visualization of processed video frames and overlayed license-plate information.  
Rendering is performed via OpenCV’s GPU-accelerated pipeline on the Jetson Nano \cite{jetson_nano_doc}.

\section{Specific Actor}
\label{sec:display_specs}
The actor device used is a standard HDMI display connected through the Jetson Nano’s HDMI 2.0 Type-A port.  
It receives RGB frames rendered in real time and displays textual overlays produced by the inference module.

\section{Library}
\subsection{Description}
The display actor is implemented in software through the OpenCV Python library, specifically its \texttt{highgui} module.  
It renders processed frames in real time and overlays text and bounding boxes on the live video stream \cite{opencv_doc_2024}.

\subsection{Installation}
The required package is installed via:
\begin{verbatim}
	sudo apt install python3-opencv
\end{verbatim}
After installation, verify using:
\begin{verbatim}
	xrandr
\end{verbatim}
to ensure HDMI detection and resolution configuration.

\subsection{Functions}
Essential OpenCV functions used:
\begin{itemize}
	\item \texttt{cv2.imshow()} – displays processed video frames.
	\item \texttt{cv2.putText()} – overlays recognition messages.
	\item \texttt{cv2.rectangle()} – draws bounding boxes on detected regions.
	\item \texttt{cv2.waitKey()} – defines frame-refresh delay.
\end{itemize}

\subsection{Data Characteristics}
\paragraph{Data Types.}  
RGB image frames represented as \texttt{uint8} arrays; UTF-8 text for overlay messages.

\paragraph{Data Structure.}  
Each frame is a NumPy array of shape (height, width, 3), over which graphical elements are rendered directly.

\paragraph{Data Quantity.}  
Approximately 25–30 frames per second are displayed, each frame about 2–3 MB (720p).  
Latency from inference to display < 100 ms \cite{edgeAIbook}.

\paragraph{Data Quality.}  
GPU-accelerated rendering maintains clarity and smoothness.  
Color-coded overlays (green = recognized, red = unregistered) enhance user perception.  
HDMI 2.0 ensures stable 60 Hz output without frame tearing.

\section{Example}
In this section, a complete example illustrates how the display actor visualizes the system response.

\subsection{Manual}
To operate, connect the HDMI cable between the Jetson Nano and monitor, select the proper input source, and power both devices.

\subsection{Inside the Example and Code}
\begin{lstlisting}[language=Python, caption={Displaying a license plate recognition message on Jetson Nano display}]
	import cv2
	
	frame = cv2.imread("car_emden.jpg")
	cv2.putText(frame, "Willkommen Fahrzeug EMD AB 1234", (40, 50),
	cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 0), 2)
	cv2.imshow("License Plate Display", frame)
	cv2.waitKey(0)
	cv2.destroyAllWindows()
\end{lstlisting}

\noindent
This demonstrates the entry event for a vehicle from Emden, Deutschland.  
When the plate “\texttt{EMD AB 1234}” is recognized, the system displays the localized message  
\textit{"Willkommen Fahrzeug EMD AB 1234"} (“Welcome Vehicle EMD AB 1234”).  
For exits, it shows  
\textit{"Ausfahrt registriert für EMD AB 1234"} (“Exit recorded for EMD AB 1234”).

\section{Calibration}
Calibration is limited to brightness, contrast, and resolution adjustments for optimal visibility.  
Settings are managed using \texttt{xrandr} or the desktop display configuration under JetPack 5.x.

\section{Simple Code}
The OpenCV example above also serves as the minimal functional code verifying proper HDMI output and text overlay generation.

\section{Simple Application}
During live inference:
\begin{itemize}
	\item \textit{Entry Event:} “Willkommen Fahrzeug EMD AB 1234” displayed in green.
	\item \textit{Exit Event:} “Ausfahrt registriert für EMD AB 1234” displayed in blue.
\end{itemize}
These localized messages provide clear user feedback at the system interface.

\section{Tests}
\subsection{Simple Function Test}
Run:
\begin{verbatim}
	xrandr
\end{verbatim}
to confirm that the HDMI display is detected and configured.

\subsection{Test all Functions}
Execute the complete inference script; verify real-time text overlays, color changes, and frame-rate stability ( greater than equal to 25 FPS).

\section{Constraints}
\begin{itemize}
	\item Requires HDMI-compatible display; analog VGA unsupported.  
	\item Minimum 25 FPS for fluid real-time feedback.  
	\item Text overlays must avoid covering detection regions.  
	\item Manual brightness/contrast tuning recommended.  
\end{itemize}
