%=== Domain Knowledge: Application (ANPR on Jetson Nano) ======================
\section{Part Domain Knowledge — Application}
\label{sec:domain-knowledge-application}

\noindent\textbf{Context.}
We deploy \emph{Automatic Number Plate Recognition (ANPR)} on an embedded edge device (NVIDIA Jetson Nano B01 + USB UVC camera) to detect plates with a real-time object detector and read text via OCR. Typical use cases include barrierless parking, campus access control, and vehicle audit trails where on-device inference reduces latency and preserves data sovereignty \cite{nvidiaJetsonNano, anprSurvey, yolov5, easyocr, edgePrivacy}.

%----------------------------- Problem Definition -----------------------------
\subsection{Problem}
Given a live RGB stream, output tuples \(\langle\)timestamp, plate\_id, bbox, confidences, plate\_text, policy\(\rangle\) with end-to-end latency \(\le\) \SI{200}{ms} at \(\ge\) \SI{15}{fps}. The system must be robust to weather (rain, snow), illumination (night, headlight glare), and pose (yaw/pitch) typical for driveway/parking entrances \cite{anprSurvey}.

%----------------------------- Data Acquisition -------------------------------
\subsection{Data Acquisition (Camera and Setup)}
\textbf{Camera:} Microsoft LifeCam Show, UVC USB 2.0, 2\,MP sensor. Operated at 640\(\times\)480 or 1280\(\times\)720 for stable fps on Nano.
\textbf{Mounting:} Rigid pole/wall, angle $10^\circ$–$25^\circ$ downward; avoid facing bright road signs. 
\textbf{Exposure:} Lock exposure/white balance where possible to reduce inter-frame variance; enable short shutter and higher gain at night to reduce motion blur.
\textbf{Ingestion:} V4L2/GStreamer to NVMM (zero-copy) where possible to keep GPU busy \cite{nvidiaJetsonNano}.

%----------------------------- Data Quantity ----------------------------------
\subsection{Data Quantity}
\begin{itemize}[leftmargin=1.1em]
	\item \textbf{Detection:} At least 5k diverse plate instances for fine-tuning (angles, distances, weather, day/night).
	\item \textbf{OCR:} Balancing samples across characters and common bigrams/trigrams (e.g., \texttt{AA}, \texttt{ER}, \texttt{MÜ}) to reduce bias \cite{ocrSurvey}.
	\item \textbf{Evaluation:} 10–20\% held-out sequences per condition (day, night, rain) and at least 1 week of continuous logs for drift checks.
\end{itemize}

%----------------------------- Data Quality -----------------------------------
\subsection{Data Quality}
Quality gates before training/inference:
\begin{enumerate}[leftmargin=1.1em]
	\item \textbf{Sharpness}: Laplacian variance \(>\) threshold; reject blurred frames.
	\item \textbf{Brightness}: Mean plate ROI within [0.2, 0.8] (normalized); cap highlights.
	\item \textbf{Plate Size}: Minimum plate height in frame and in crop; upscale only within 1.5\(\times\) to limit artifacts.
	\item \textbf{Label Consistency}: IoU agreement \(>\)0.5 across annotators for detection; OCR string validator (regex for country format).
\end{enumerate}

%----------------------------- Relevance --------------------------------------
\subsection{Data Relevance (Formats \& Policies)}
German/EU plates follow regulated layouts (country band, district code, letters–digits, inspection badge positions). We encode format-aware validators and country-specific regex to improve OCR post-processing and reduce false accepts \cite{anprSurvey, euPlateStd}.

%----------------------------- Notes on Privacy -------------------------------
\subsection{Privacy and Policy Notes (GDPR-aware)}
Perform on-device inference; store only hashed plate strings plus salted last-4 preview where policy allows; enforce minimal retention and role-based access; display site signage and document purpose/legitimate interest \cite{gdpr, edgePrivacy}.

%----------------------------- What we will measure ---------------------------
\subsection{Operational KPIs}
\begin{itemize}[leftmargin=1.1em]
	\item Detection AP\(_{50}\) on held-out day/night; OCR CER/WER on plate crops.
	\item End-to-end latency (camera \(\rightarrow\) decision), fps stability over \SI{30}{min}.
	\item False accept / false reject on whitelist; drift rate vs. weather/time-of-day.
\end{itemize}
