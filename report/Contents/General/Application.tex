\section{Application Domain Knowledge}

\subsection{Domain Knowledge — Application Domain}
The proposed system lies in the domain of \textbf{Smart Mobility} and \textbf{Airport Traffic Automation}, specifically focusing on \textbf{Edge-Based License Plate Detection and Parking Management}.  
Taking the example of an airport deployment, at the entry and exit gates, the system's camera interface is able to detects license plates and record timestamps. The time between these time-stamps is subsequently computed. If the duration exceeds a predefined threshold (10 minutes), the system further calculates a parking charge based on the time of stay.

The system uses the \textbf{NVIDIA Jetson Nano} and is able to operate offline keeping an internet connection optional. It uses \textbf{YOLOv8} in order to detect license plates, \textbf{EasyOCR} is used to recognize characters within the plate, and an \textbf{SQLite} database is used for storing tracked timestamps. This enables real-time monitoring and local billing without the use of a cloud based infrastructure \cite{anpr-survey-2020, easyocr-2020, nvidia-jetson}.

\vspace{0.5em}
\noindent\textbf{Need for Edge Based Systems}  
Environments such as airports and parking lots operate continuously. Here, minimizing latency and securely processing local data are ensured using Edge \ac{AI}:
\begin{itemize}
	\item Operation even during network outages.
	\item Low latency ($\approx$15–25 FPS) suitable for vehicle flow.
	\item \ac{GDPR}-compliant local storage.
	\item Lesser costs compared to cloud-based \ac{ANPR} implementations.
\end{itemize}

\subsection{Domain Application Specific Problem}
In the context of airports, there is high vehicular density near departure and arrival stations where passengers enter and exit.  
Security rules usually allow short stays (e.g., under 10 minutes). Manual enforcement using human officers or ticket machines is inefficient and inconsistent.  
The system solves the problem of manual logging by:
\begin{itemize}
	\item Being able to detect and recognize license plates automatically. 
	\item Generating entry and exit timestamps, enabling  the computation of \textbf{time difference}. 
	\item Triggering a \textbf{fee} event when stay duration exceeds a predefined time limit.
\end{itemize}

\subsection{Data Acquisition}
\textbf{Operational data:} Continuous video stream from a single fixed camera at the airport gate (1080p, 25 FPS).  
\textbf{Data pipeline:} \textbf{YOLOv8} detects license plate bounding boxes → \textbf{EasyOCR} extracts alphanumeric text → timestamps are stored in SQLite as either \emph{entry} or \emph{exit} events.  
The same camera detects both, using logic to compare time differences.  
\textbf{Training/validation data:} Public datasets (CCPD, OpenALPR) plus airport-captured samples to adapt to real lighting, glare, and plate types \cite{ccpd-2018, openalpr-benchmark, eu-anpr}.

\subsection{Data Quantity}
\begin{itemize}
	\item Training dataset size: $\sim$2000 images combining open datasets and airport footage.
	\item Runtime: real-time detection at 20–25 FPS.
	\item License plates are identified and corresponsing timestamps recorded — \emph{entry} and \emph{exit} — further logged in internal SQLite database.
	\item SQLite handles up to $\sim$10$^6$ records with date, time, and license ID.
	\item Historical data exported periodically for audit and analytics.
\end{itemize}

\subsection{Data Quality}
Quality determines billing precision:
\begin{itemize}
	\item \textbf{Preprocessing:} \ac{CLAHE} contrast enhancement and denoising before \ac{OCR}.
	\item \textbf{Confidence gating:} Accept \textbf{YOLO} detections only when confidence $>0.6$.
	\item \textbf{Time consistency:} Duplicate detections within 5 seconds are suppressed.
	\item \textbf{Regex validation:} Only valid EU/German plate formats are stored.
	\item \textbf{Lighting calibration:} Camera auto-gain and exposure tuned for day/night operation.
\end{itemize}

\subsection{Data Relevance}
All collected data directly supports the time difference computation task:
\begin{enumerate}
	\item Valid license plates are recorded, events (entry/exit) are logged. Invalid frames are not logged.
	\item Events are timestamped precisely (milliseconds) for accurate dwell-time calculation.
	\item Region-specific plate formats ensure relevance to local regulations.
	\item Logs are stored locally for privacy and regulatory compliance.
\end{enumerate}

\subsection{Outliers}
Outliers refer to detections that do not follow the normal entry-exit pattern:
\begin{itemize}
	\item Vehicles detected multiple times without leaving (false multiple exits).
	\item Blurred plates or partially visible text.
	\item Vehicles without visible plates (blocked or dirty).
\end{itemize}
\textbf{Mitigation:} System uses timestamp grouping and last-seen confidence to identify invalid sequences and exclude them from billing.

\subsection{Anomalies}
Anomalies indicate irregular operational conditions:
\begin{itemize}
	\item \textbf{Double detections:} Same vehicle logged twice within seconds.
	\item \textbf{Exit-before-entry:} Clock or logic errors (handled with checks).
	\item \textbf{\ac{OCR} mismatch:} Entry and exit plates differ by 1–2 characters (e.g., due to reflection).
\end{itemize}
\textbf{Resolution:} Pattern matching and fuzzy string similarity (\texttt{difflib}) handle small \ac{OCR} variations before billing logic is triggered.

\subsection*{Summary}
Application domain wise, the solution integrates \textbf{computer vision}, \textbf{edge \ac{AI}}, and \textbf{local data management} aiding in the automation of vehicle monitoring within high density environments such as airports. 
A single Jetson Nano-based camera recognizes both \textbf{entry and exit} events, records timestamps, calculates \textbf{dwell time}, and automatically triggers billing for vehicles exceeding the permitted 10-minute duration.  
The automated workflow enables automatic logging and ticketing with minimal human monitoring, leading to an improvement in \textbf{operational efficiency} and \textbf{reliability}.

In terms of data, high \textbf{data quality} is ensured through real-time validation, confidence-based filtering, and local storage.  
It adheres to \textbf{\ac{GDPR}-compliant practices} by processing and storing data directly on the Jetson Nano without cloud transmission.  
An edge-first system reduces latency, promotes low maintenance costs, and ensures that the system is functional in environments that are restricted in terms of network or internet availability.

Beyond airport use, the developed framework can be scaled for other domains such as:
\begin{itemize}
	\item \textbf{Smart parking lots} for retail, hospitals, or universities.  
	\item \textbf{Logistics hubs} for tracking delivery vehicle dwell times.  
	\item \textbf{Security Facilities} (especially government embassy areas, military facilities) where access control and logging is important.  
\end{itemize}

In addition, integration such as \textbf{Vehicle-to-Infrastructure (V2I)} and \textbf{Vehicle-to-Vehicle (V2V)} communication systems can translate into mobile and connected modules.  
Extensions in this form promote dynamic traffic coordination, \textbf{availability broadcasting at entry and exit gates in real time}, and security alerts automated — defining components that could prove vital to future infrastructures such as smart airports.  

In summary, the solution describes lightweight embedded \ac{AI} that is able to deliver automation that is promotes social, operational, and economic benefits, aiding in the evolution of  \textbf{autonomous, intelligent, and sustainable transportation ecosystems} \cite{smart-parking, privacy-edge, nvidia-jetson}.
