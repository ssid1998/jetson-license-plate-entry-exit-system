\section{Application Domain Knowledge}

\subsection{Domain Knowledge — Application Domain}
The proposed system lies in the domain of \textbf{Smart Mobility} and \textbf{Airport Traffic Automation}, specifically focusing on \textbf{Edge-Based License Plate Detection and Parking Management}.  
It uses a single fixed camera at the airport gate to automatically detect the entry and exit of vehicles, record timestamps, and calculate \textbf{dwell time}. If the stay exceeds a predefined threshold (10 minutes), the system automatically computes and logs a \textbf{parking charge}.

Deployed on an \textbf{NVIDIA Jetson Nano}, the system operates completely offline, combining \textbf{YOLOv8} for plate detection, \textbf{EasyOCR} for recognition, and a lightweight \textbf{SQLite} database for event tracking.  
This allows real-time monitoring, instant detection, and local billing without any need for cloud infrastructure \cite{anpr-survey-2020, easyocr-2020, nvidia-jetson}.

\vspace{0.5em}
\noindent\textbf{Why edge?}  
The airport environment demands continuous operation, minimal latency, and secure local data handling. Edge AI ensures:
\begin{itemize}
	\item Operation even during network outages.
	\item Low latency ($\approx$15–25 FPS) suitable for vehicle flow.
	\item GDPR-compliant local storage.
	\item Lower operating costs compared to cloud-connected ANPR systems.
\end{itemize}

\subsection{Application Problem}
At airports, vehicles frequently stop near terminals to pick up or drop passengers.  
Security rules usually allow short stays (e.g., under 10 minutes). Manual enforcement using human officers or ticket machines is inefficient and inconsistent.  
Our system eliminates manual logging by:
\begin{itemize}
	\item Automatically detecting and recognizing plates at both entry and exit using a \textbf{single camera}.
	\item Computing the \textbf{time difference} between entry and exit for each vehicle.
	\item Automatically triggering a \textbf{charge} when the stay exceeds 10 minutes.
\end{itemize}
This enables efficient, unattended management of short-term parking and drop-off areas.

\subsection{Data Acquisition}
\textbf{Operational data:} Continuous video stream from a single fixed camera at the airport gate (1080p, 25 FPS).  
\textbf{Data pipeline:} YOLOv8 detects license plate bounding boxes → EasyOCR extracts alphanumeric text → timestamps are stored in SQLite as either \emph{entry} or \emph{exit} events.  
The same camera detects both, using logic to compare time differences.  
\textbf{Training/validation data:} Public datasets (CCPD, OpenALPR) plus airport-captured samples to adapt to real lighting, glare, and plate types \cite{ccpd-2018, openalpr-benchmark, eu-anpr}.

\subsection{Data Quantity}
\begin{itemize}
	\item Training dataset size: $\sim$20k images combining open datasets and airport footage.
	\item Runtime: real-time detection at 20–25 FPS.
	\item Each vehicle generates 2 records — \emph{entry} and \emph{exit} — stored in SQLite.
	\item SQLite handles up to $\sim$10$^6$ records with date, time, and license ID.
	\item Historical data exported periodically for audit and analytics.
\end{itemize}

\subsection{Data Quality}
Quality determines billing precision:
\begin{itemize}
	\item \textbf{Preprocessing:} CLAHE contrast enhancement and denoising before OCR.
	\item \textbf{Confidence gating:} Accept YOLO detections only when confidence $>0.6$.
	\item \textbf{Time consistency:} Duplicate detections within 5 seconds are suppressed.
	\item \textbf{Regex validation:} Only valid EU/German plate formats are stored.
	\item \textbf{Lighting calibration:} Camera auto-gain and exposure tuned for day/night operation.
\end{itemize}

\subsection{Data Relevance}
All collected data directly supports the parking time computation task:
\begin{enumerate}
	\item Only valid plate events (entry/exit) are recorded; irrelevant frames are discarded.
	\item Events are timestamped precisely (milliseconds) for accurate dwell-time calculation.
	\item Region-specific plate formats ensure relevance to local regulations.
	\item Logs are stored locally for privacy and regulatory compliance.
\end{enumerate}

\subsection{Outliers}
Outliers refer to detections that do not follow the normal entry-exit pattern:
\begin{itemize}
	\item Vehicles detected multiple times without leaving (false multiple exits).
	\item Blurred plates or partially visible text.
	\item Vehicles without visible plates (blocked or dirty).
\end{itemize}
\textbf{Mitigation:} System uses timestamp grouping and last-seen confidence to identify invalid sequences and exclude them from billing.

\subsection{Anomalies}
Anomalies indicate irregular operational conditions:
\begin{itemize}
	\item \textbf{Double detections:} Same vehicle logged twice within seconds.
	\item \textbf{Exit-before-entry:} Clock or logic errors (handled with checks).
	\item \textbf{OCR mismatch:} Entry and exit plates differ by 1–2 characters (e.g., due to reflection).
\end{itemize}
\textbf{Resolution:} Pattern matching and fuzzy string similarity (\texttt{difflib}) handle small OCR variations before billing logic is triggered.

\subsection*{Summary}
The application domain integrates \textbf{computer vision}, \textbf{edge AI}, and \textbf{local data management} to automate vehicle monitoring within airport terminals.  
A single Jetson Nano-based camera recognizes both \textbf{entry and exit} events, records timestamps, calculates \textbf{dwell time}, and automatically triggers billing for vehicles exceeding the permitted 10-minute duration.  
This fully automated workflow eliminates the need for manual ticketing or human monitoring, improving \textbf{efficiency}, \textbf{accuracy}, and \textbf{24/7 operational reliability}.

From a data perspective, the system maintains high \textbf{data quality} through real-time validation, confidence-based filtering, and local persistence of relevant information only.  
It adheres to \textbf{GDPR-compliant practices} by processing and storing data directly on the Jetson Nano without cloud transmission.  
This edge-first approach minimizes latency, reduces maintenance costs, and ensures continued functionality even in network-restricted environments.

Beyond airport use, the developed framework can be scaled for other domains such as:
\begin{itemize}
	\item \textbf{Smart parking lots} for retail, hospitals, or universities.  
	\item \textbf{Logistics hubs} for tracking delivery vehicle dwell times.  
	\item \textbf{High-security facilities} (embassies, military zones) where controlled access logging is critical.  
\end{itemize}

Furthermore, integration with \textbf{Vehicle-to-Infrastructure (V2I)} and \textbf{Vehicle-to-Vehicle (V2V)} communication systems can transform this into a connected mobility module.  
Such extensions will enable dynamic traffic coordination, \textbf{real-time gate availability broadcasting}, and automated security alerts — key components of next-generation \textbf{Smart Airport Infrastructure}.  

Overall, the proposed solution demonstrates how lightweight embedded AI can deliver high-impact automation with measurable social, operational, and economic benefits, supporting the transition toward \textbf{autonomous, intelligent, and sustainable transportation ecosystems} \cite{smart-parking, privacy-edge, nvidia-jetson}.
