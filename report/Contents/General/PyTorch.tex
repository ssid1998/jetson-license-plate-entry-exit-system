\section{PyTorch}

\index{PyTorch}

% --------------------------------------------------------
\subsection{Introduction}
\index{PyTorch!Introduction}

\noindent
PyTorch is an open-source deep learning framework. It implements an imperative (define-by-run) execution model that ensures flexibility in neural network construction and debugging during runtime \cite{paszke2019pytorch}. PyTorch provides a powerful tensor library augmented with GPU acceleration via the \texttt{autograd} engine, that is able to provide a computational framework for gradients in  optimization tasks for deep learning workloads \cite{pytorchdocs}.

PyTorch distinguishes itself from previous deep learning frameworks in its dynamic computation graph design, That is able to construct a computational graph during execution and utilizes standard Python control flow for defining models \cite{goodfellow2016deep}. Apart from its fundamental tensor and gradient capabilities, PyTorch provides a variety of modules  pertaining to neural network layers, loss functions, optimization algorithms, and efficient data pipelines. Such  component support makes PyTorch a versatile and scalable framework for deep learning tasks and embedded system deployment \cite{pinto2022springer}.

In the context of this project, PyTorch is at the core deep learning framework responsible for the execution of license plate detection model on the Jetson Nano. Following OpenCV image capture and processing, PyTorch is responsible for:

\begin{itemize}
	\item Transforming input frames into tensors suited towards neural network inference,
	\item Running the optimized deep learning model on the Jetson Nano’s CUDA-enabled GPU, and
	\item Interfacing with \texttt{torchvision} utilities for model loading, tensor transformations, and streamlined inference workflows.
\end{itemize}

This tight coupling between PyTorch and the Jetson platform is instrumental in building an efficient pipeline for identifying license plates and logging detection results within the system \cite{jetsonpytorch}.

% --------------------------------------------------------
\subsection{Description}
\index{PyTorch!Description}

PyTorch comprises of modular components that support deep learning model development, training, and deployment. Each module is responsible for a specific function. The most relevant components are described as follows:

\begin{itemize}
	\item \texttt{torch}: The core package responsible for providing tensor operations and mathematical functions.
	\item \texttt{torch.nn}: Module consisting of neural network layers, activation functions, and loss functions.
	\item \texttt{torch.optim}: Responsible for the implementation of optimization algorithms, like SGD and Adam, for neural network training.
	\item \texttt{torchvision}: Provides tools pertaining to image-based tasks, including datasets and pre-trained models.
	\item \texttt{torch.utils.data}: Contains data-loading utilities for dataset abstraction and data loading.
	\item \texttt{autograd}: Enables gradient computation for backpropagation.
\end{itemize}

PyTorch and its dynamic computation graph enables rapid experimentation with custom neural network architectures, often vital in computer vision tasks \cite{paszke2019pytorch}. The integration with Python and popular scientific libraries such as NumPy and SciPy augments data manipulation and pre-processing \cite{goodfellow2016deep}.

In addition, PyTorch offers a diverse ecosystem of vision-specific utilities through \texttt{torchvision} library. It provides pre-built datasets, common image transforms, and pre-trained models such as ResNet, VGG, and YOLO that can be customized \cite{torchvision_docs}. PyTorch also supports mixed-precision training, distributed computing, and optimized CUDA kernels, enabling efficient training and inference \cite{nvidia_jetson}.

\subsection{Installation}
\index{PyTorch!Installation}

PyTorch installation starts with the preparation of a standard Python environment. It is typically installed using package managers like as \texttt{pip} or \texttt{conda}. Core dependencies include NumPy, SciPy, and Python development headers. For GPU acceleration, CUDA and cuDNN libraries are essential \cite{paszke2019pytorch, kulkarni2022cvprojects}. In common workflows, virtual environments may be created using \texttt{venv} or \texttt{conda}. Once activated, PyTorch is then installed along with optional packages such as \texttt{torchvision}, \texttt{torchaudio}, or \texttt{torchtext} completing the deep learning stack. 

An installation procedure for Jetson Nano includes:

\begin{itemize}
	\item \textbf{Install NVIDIA JetPack SDK:} Download and install the JetPack SDK.
	\item \textbf{Set up Python environment:} Install Python 3 and \texttt{pip}. A virtual environment using \texttt{virtualenv} or \texttt{conda} can further be created to isolate dependencies.
	\item \textbf{Install PyTorch wheel:} Download the pre-built PyTorch wheel specifically optimized for Jetson devices and install it using \texttt{pip}.
	\item \textbf{Install supporting libraries:} Install \texttt{torchvision} ensuring CUDA support is enabled for GPU-accelerated operations.
	\item \textbf{Verify installation:} Execute a simple tensor operation and ensure GPU availability to confirm successful PyTorch operation.
\end{itemize}

The following command installs the core \PYTHON{torch} library along with \PYTHON{torchvision} and \PYTHON{torchaudio} modules. The official PyTorch documentation provides detailed installation instructions and platform-specific wheels \cite{paszke2019pytorch, pytorchdocs}.


\begin{tcolorbox}[colback=white,colframe=black,coltext= ShellColor, sharp corners, boxrule=0.8pt]
	\begin{small}
		\begin{verbatim}
			
			pip install torch torchvision torchaudio	\end{verbatim}
	\end{small}
\end{tcolorbox}

\subsection{Example -- Description}
\index{PyTorch!Example Description}

The example script \FILE{PyTorchLPExample.py} demonstrates a minimal but representative PyTorch workflow for licence plate detection on the Jetson Nano. Its purpose is to verify that:
\begin{itemize}
	\item the camera is correctly connected and accessible,
	\item OpenCV can capture and display frames in real time,
	\item PyTorch tensors can be created from camera frames and moved to GPU if available, and
	\item basic model inference and frame annotation operations work as expected on the Jetson Nano.
\end{itemize}

In essence, the example performs the following steps:
\begin{enumerate}
	\item Starts the default camera device using \PYTHON{cv2.VideoCapture(0)}.
	\item Reads and processes frames in a loop, while checking the validity of returned frames.
	\item Transforms frames into PyTorch tensors using \PYTHON{torchvision.transforms.ToTensor()}.
	\item Executes a PyTorch model (dummy or pre-trained) on the tensor to enable licence plate detection.
	\item Annotates the original frame with predicted bounding boxes and confidence scores using \PYTHON{cv2.rectangle} and \PYTHON{cv2.putText}.
	\item Displays annotated frames in a window using \PYTHON{cv2.imshow}.
	\item Terminates loops when specific keys are pressed (e.g., \texttt{q}), thereby releasing resources.
\end{enumerate}

\subsection{Example -- Manual}
\index{PyTorch!Example Manual}

The following instructions describe how to run the PyTorch example on the Jetson Nano:

\begin{itemize}
	\item \textbf{Step 1:} Ensure that the Jetson Nano is powered on and that the camera (CSI or USB) is connected correctly as configured for the project.
	\item \textbf{Step 2:} Activate the Python virtual environment (if used) that contains the project dependencies, including PyTorch and torchvision.
	\item \textbf{Step 3:} Navigate to the project directory where \FILE{PyTorchLPExample.py} is stored, for example:
	
	\begin{tcolorbox}[colback=white, colframe=black, coltext=ShellColor, sharp corners, boxrule=0.8pt]
		\begin{small}
			\begin{verbatim}
				cd ~/projects/license-plate-detection/pytorch_tests
			\end{verbatim}
		\end{small}
	\end{tcolorbox}
	
	\item \textbf{Step 4:} Run the example script:
	
	\begin{tcolorbox}[colback=white, colframe=black, coltext=ShellColor, sharp corners, boxrule=0.8pt]
		\begin{small}
			\begin{verbatim}
				python PyTorchLPExample.py
			\end{verbatim}
		\end{small}
	\end{tcolorbox}
	
	\item \textbf{Step 5:} A window should open showing the camera stream with any model predictions (dummy bounding boxes or real detections) overlaid. Verify that the frame rate is smooth and that the bounding boxes are correctly displayed.
	\item \textbf{Step 6:} Press the configured exit key (e.g., \texttt{q}) to close the window, release the camera, and terminate the program.
\end{itemize}

This procedure provides a quick check of camera connectivity, PyTorch model inference, and real-time annotation before running the full Jetson Nano licence plate detection pipeline.
