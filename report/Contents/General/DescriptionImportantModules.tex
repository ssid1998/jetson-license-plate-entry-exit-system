\section{Introduction}
\index{Modules!Introduction}

\noindent
Modern computer vision systems increasingly rely on the integration of efficient image-processing libraries, flexible deep learning frameworks, and powerful learning algorithms to achieve accurate and real-time performance. In this project, \textbf{OpenCV}, \textbf{PyTorch}, and \textbf{Convolutional Neural Networks (CNNs)} form the core technological foundation for implementing a real-time license plate detection system on the NVIDIA Jetson Nano B01. Together, these components provide a complete end-to-end pipeline that spans image acquisition, preprocessing, deep learning inference, and result visualization.

At the lowest level of the pipeline, \textbf{OpenCV (Open Source Computer Vision Library)} serves as the primary image and video processing framework. OpenCV is an open-source library originally developed by Intel and now maintained by OpenCV.org, offering a wide range of optimized algorithms for real-time computer vision applications \cite{Bradski:2000,opencv_wiki}. It provides essential functionality for camera interfacing, frame capture, color space conversion, resizing, and visual annotation of detection results. Due to its highly optimized implementation and support for hardware acceleration, OpenCV is particularly well suited for embedded platforms such as the NVIDIA Jetson Nano, where computational resources are limited but real-time performance is required \cite{opencv_lfs,opencv_on_wheels}. In this project, OpenCV acts as the visual backbone that bridges raw camera input with deep learning inference.

Building upon this image-processing layer is \textbf{PyTorch}, a modern, open-source deep learning framework designed to offer an intuitive Python interface while maintaining high computational efficiency. PyTorch employs a dynamic, define-by-run execution model that allows neural networks to be constructed and modified during runtime, facilitating rapid experimentation and debugging \cite{paszke2019pytorch}. At its core, PyTorch provides a powerful tensor library with GPU acceleration and automatic differentiation through its \texttt{autograd} engine, enabling efficient gradient computation for deep learning workloads \cite{pytorchdocs}. In this system, PyTorch is responsible for converting pre-processed image frames into tensors, executing the trained detection model on the Jetson Nano’s CUDA-enabled GPU, and managing inference workflows through supporting libraries such as \texttt{torchvision} \cite{jetsonpytorch}. This tight integration allows the system to perform real-time inference while maintaining flexibility and scalability.

At the algorithmic core of the detection pipeline are \textbf{Convolutional Neural Networks (CNNs)}, which form the dominant deep learning architecture for image-based tasks. CNNs are specifically designed to process data with a grid-like structure, such as images, by exploiting local spatial correlations through convolutional layers and parameter sharing \cite{lecun1998gradient,goodfellow2016deep}. By hierarchically extracting features ranging from simple edges and textures to high-level semantic patterns, CNNs enable robust object detection under varying lighting conditions, viewing angles, and background clutter \cite{krizhevsky2012imagenet,simonyan2014very}. In the context of license plate detection, CNNs learn discriminative visual features that allow accurate localization and classification of plates in real time, even under challenging environmental conditions \cite{pinto2022springer}.

Together, OpenCV, PyTorch, and CNNs form a cohesive and efficient processing pipeline: OpenCV handles image acquisition and preprocessing, PyTorch provides the computational framework for model execution, and CNNs supply the representational power required for reliable visual detection. This modular yet tightly integrated architecture enables real-time computer vision on embedded hardware and provides a scalable foundation for applications in traffic monitoring, security, and intelligent transportation systems \cite{jetsonpytorch,nvidia_jetson}. The following chapters examine each of these components in detail, outlining their structure, installation, and role within the overall system.
